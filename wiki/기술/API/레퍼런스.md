# API 레퍼런스

> 한 줄 요약: Polymarket의 공개 API 엔드포인트와 실무 예제를 정리한다. (curl, Python requests, Node fetch)

## 내용

주요 엔드포인트: 이벤트 목록, 마켓 상세, 프로필, 리더보드(leaderboard), 마켓 리스트(markets). 본 문서는 copytrade 전략 구현자가 API를 사용해 리더보드와 마켓 상태를 자동으로 수집하고, 간단한 재시도/타임아웃 처리를 포함한 예제 코드를 통해 신뢰성 있게 데이터를 가져오는 방법을 설명한다.

### 권장 사용 패턴

- 요청 타임아웃과 재시도(backoff)를 구현하여 rate limit/네트워크 실패에 강하게 만든다.
- 중요한 수치(예: 리더보드 상위 N명의 포지션)는 원시 응답을 로컬에 보관(raw/fetched/)하여 재검증이 가능하게 한다.
- API 호출 빈도는 rate limit을 고려해 throttling한다.

## 예제

### 1) curl (간단 호출)

- 리더보드 상위 10명 가져오기

```bash
curl -s "https://clob.polymarket.com/rewards/leaderboard?limit=10" | jq '.'
```

- 활성 이벤트 목록

```bash
curl -s "https://gamma-api.polymarket.com/events?limit=20&active=true" | jq '.events'
```

### 2) Python (requests)

```python
import requests
from requests.adapters import HTTPAdapter
from urllib3.util.retry import Retry

session = requests.Session()
retries = Retry(total=3, backoff_factor=0.5, status_forcelist=[429,500,502,503,504])
session.mount('https://', HTTPAdapter(max_retries=retries))

def fetch_events(limit=20):
    url = f"https://gamma-api.polymarket.com/events?limit={limit}&active=true"
    resp = session.get(url, timeout=5)
    resp.raise_for_status()
    data = resp.json()
    # 예: 이벤트 id와 제목 추출
    return [(e['id'], e.get('title')) for e in data.get('events', [])]

if __name__ == '__main__':
    events = fetch_events(20)
    print(events)
```

설명: 세션 재사용과 Retry를 통해 안정성을 높였다. 응답 스키마는 변경될 수 있으므로 필드 접근 시 안전하게 예외처리한다.

### 3) Node.js (fetch)

```js
import fetch from 'node-fetch';

async function fetchMarkets(limit = 50) {
  const url = `https://gamma-api.polymarket.com/markets?limit=${limit}`;
  const res = await fetch(url, { timeout: 5000 });
  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  const data = await res.json();
  return data.markets || data;
}

fetchMarkets(20).then(m => console.log(m.slice(0,3))).catch(console.error);
```

## 응용 예제: 리더보드 기반 트레이더 식별

1. 리더보드 API로 상위 N명 주소 수집
2. 각 주소로 프로필/포지션 API 호출
3. 최근 포지션 패턴(예: 베팅 빈도, 평균 포지션 크기)을 로컬 DB에 집계
4. 자동화된 copytrade 대상 선정 로직에 반영

(구현 시 rate limit과 API 응답 지연을 반드시 고려)
