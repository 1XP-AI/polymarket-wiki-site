# Polymarket 개요

> 한 줄 요약: Polymarket 플랫폼의 핵심 구조, 개념(이벤트/마켓/토큰)과 데이터/오더북 구성 요약.

## 개요

Polymarket은 **세계 최대의 예측시장(Prediction Market)** 플랫폼으로, Polygon 블록체인 위에서 운영된다. 사용자가 실제 세계 이벤트의 결과에 USDC.e를 사용하여 베팅하며, 모든 정산은 온체인에서 이루어진다.

## 핵심 구조: 3개 API

| API | Base URL | 용도 | 인증 |
|-----|----------|------|------|
| **Gamma API** | `gamma-api.polymarket.com` | 마켓/이벤트/프로필/검색 | 불필요 (Public) |
| **Data API** | `data-api.polymarket.com` | 포지션/트레이드/활동/리더보드 | 불필요 (Public) |
| **CLOB API** | `clob.polymarket.com` | 오더북/가격/주문/거래 | 읽기: Public / 쓰기: L1/L2 Auth |

### SDK

- TypeScript: `@polymarket/clob-client`
- Python: `py-clob-client`
- Rust: `polymarket-rs`

### Copytrade 관점에서의 API 선택

- **트레이더 추적**: Data API (`/v1/leaderboard`)로 상위 지갑 확인 → Gamma API로 해당 지갑의 이벤트/마켓 매핑
- **실시간 복제**: CLOB API WebSocket으로 오더북 구독 → 주문 체결 감지
- **마켓 발견**: Gamma API의 `?active=true` 필터로 신규/활성 마켓 스캔

## 이벤트와 마켓

- **이벤트**: 하나의 질문/주제 (예: "2024 미국 대선 승자는?")
- **마켓**: 이벤트 내 개별 결과 옵션 (예: "트럼프 당선" / "바이든 당선")
- 하나의 이벤트에 여러 마ket이 존재

Gamma API 응답에서 확인되는 주요 필드:
```json
{
  "id": "이벤트 UUID",
  "ticker": "이벤트 티커",
  "title": "Full title",
  "slug": "url-friendly_slug",
  "description": "설명",
  "volume": "누적 거래량",
  "liquidity": "현재 유동성",
  "resolutionSource": "결과 판정 출처"
}
```

## 가격과 오더북

- 가격 범위: **$0.00 ~ $1.00** (확률을 나타냄)
- $0.60 = 60% 확률이라는 시장 합의
- **[[개념/CLOB]]** (Central Limit Order Book): P2P 매칭 방식
- Bid(매수)와 Ask(매도)가 호가로 쌓임
- **Spread** = Best Ask - Best Bid (호간 격차)

### 시장 충격과 슬리피지

대규모 주문은 호가 깊이를 빠르게 소진하며, 이는 [[개념/Negative-Risk]] 마켓에서 더 두드러진다. Copytrade 복제 시 원주문의 10~30% 수준으로 분할 진입하는 것이 권장된다.

## 포지션과 토큰

### CTF 기반 토큰

베팅 시 [[개념/CTF]] (Conditional Token Framework) 토큰을 받음:
- **Yes 토큰** / **No 토큰** 쌍으로 존재
- 결과가 맞으면 $1.00으로 정산, 틀리면 $0.00

### 토큰 생명주기

1. **Split**: USDC.e 입금 → Yes/No 토큰 쌍으로 분할
2. **Merge**: Yes/No 토큰 쌍을 다시 USDC.e로 합성
3. **Redeem**: 승리 토큰 → USDC.e 인출

## 주문 생명주기

| 단계 | 설명 |
|------|------|
| 1. 생성(Create) | 서명 후 CLOB에 제출 (L1/L2 인증) |
| 2. 스코어링(Scoring) | 오더북 점수 산정 후 등록 |
| 3. 매칭(Matching) | 가격-시간 우선 순위로 상대 주문과 매칭 |
| 4. 체결(Fill) | 온체인 실행, 토큰 이동 |
| 5. 정산(Settlement) | 마켓 종료 후 UMA 오라클 기반 결과가 $1/$0 정산 |

## 마켓 결과 판정

- **UMA 오라클** 사용 — 탈중앙화된 결과 판정
- 결과 공표 후: 승리 토큰 → $1.00, 패배 토큰 → $0.00
- **이의 제기(Dispute)** 가능 — UMA 투표 절차

## 수수료 구조

| 역할 | 수수료 | 비고 |
|------|--------|------|
| Taker | 수수료 있음 | 가격에 따라 차등 |
| Maker | **리베이트** 받음 | Maker Rebates Program |
| Gas | 무료 | Gasless Transaction 지원 |

## Negative Risk 마켓

[[개념/Negative-Risk]]: 다중 결과 이벤트에서 자본 효율적으로 포지션 구성 가능

- **Neg-Risk 구조**: 일반 마켓과 달리 CTF Split/Merge 없이 독립 Yes/No 마ket 여러 개 운영
- 일반 마켓 대비 필요한 자본이 줄어듦
- Polymarket의 Neg-Risk 마켓에서는 가격 불균형($\sum P(Yes) \neq 1$)이 실시간으로 교정되지만, 뉴스 직후 일시적 기회 발생

## 데이터 리소스

### WebSocket 채널

| 채널 | 유형 | 내용 |
|------|------|------|
| Market Channel | Public | 실시간 오더북, 가격 변동 |
| User Channel | Authenticated | 개인 포지션/주문 상태 |
| Sports Channel | Public | 스포츠 이벤트 전용 |
| RTDS | Public | 실시간 코멘트, 크립토 가격 |

### Subgraph

- GraphQL로 온체인 이벤트, 토큰 이동, 정산 기록 조회

## Copytrade 관점

Polymarket의 공개된 리더보드(Data API)와 실시간 WebSocket 데이터를 결합하면 상위 트레이더의 진입/청산 패턴을 실시간으로 감지할 수 있다. 이를 [[개념/Copytrade]] 전략에 통합하면 다음 이점을 얻을 수 있다:

1. **시전 타이밍**: WebSocket 체결 데이터로 원주문 감지 → 수초 내 복제
2. **유동성 검증**: 진입 전 호가 깊이 검사로 슬리피지 예측
3. **Neg-Risk 활용**: 다중 결과 마켓에서 원 트레이더의 포지션을 헤지
