# 프로그램 개요

## 요약

<!-- para: para_001 -->
> autowiki-mlx 프로젝트의 목적, 위키 구조, 사이클 기반 자율 확장 프로토콜 요약

<!-- para: para_002 -->
이 프로젝트는 **Polymarket 중심 예측시장 위키**를 자동으로 확장하는 에이전트 프로토콜을 정의한다. 핵심 질문은 다음과 같다:

> "Polymarket에서 누구를 따라 거래할 것인가, 어떻게 리스크를 관리할 것인가"

구체적인 목표:

<!-- para: para_003 -->
위키는 고정 주기로 자율 확장한다. 각 사이클에서 Writer 에이전트는 다음 작업 유형 중 하나를 선택해 실행:

| 작업 유형 | 코드 | 설명 |
|-----------|------|------|
| 문서 심화 | Deepen | 기존 얇은 문서에 예제/코드/새 출처 추가 |
| 새 문서 생성 | Expand | _health.md의 누락 개념, 빈 링크 대상 문서화 |
| 연결 강화 | Link | 문서 간 교차 링크 삽입, 관련 문서 섹션 업데이트 |
| 산출물 생성 | Output | FAQ, 보고서, 입문 가이드 → output/ |
| 외부 데이터 수집 | Fetch | DuckDuckGo, Polymarket API 등 → raw/fetched/ |

각 사이클 종료 후:

<!-- para: para_005 -->
위키 건강도(`_health.md`)는 다음 기준으로 측정:

- **문서 수**: 목표 100개 (현재 82개)
- **총 단어 수**: 목표 400,000단어 (현재 37,248)
- **평균 문서 깊이**: 목표 2,000단어/문서 (현재 454단어)
- **링크 밀도**: 목표 문서당 5개+ (현재 9.6)
- **출처 비율**: 목표 50%+ (현재 79.3% = 65/82)
- **빈 링크**: 0개 유지

Quality Score 계산식:

= min(docs/100,1)×15 + min(words/400000,1)×15
  + max(0,1-dead_links/10)×15 + min(avg_links/5,1)×15
  + min(avg_words/2000,1)×15 + (cited/docs×100)×0.25

현재 Quality Score: **67/100**

<!-- para: para_006 -->
- **중복 방지**: 새 파일 생성 전 `find wiki/ -name '파일명.md'`로 중복 확인. 1개 이상 나오면 기존 파일 확장
- **네이밍**: 폴더명 한글만, wiki/ 루트에 파일 두지 않음
- **출처 필수**: `sources:` frontmatter가 없으면 truth_score 0
- **수정 금지 파일**: wiki-writer.md, wiki-reviewer.md, wiki-program.md, autowiki-loop.sh 등
