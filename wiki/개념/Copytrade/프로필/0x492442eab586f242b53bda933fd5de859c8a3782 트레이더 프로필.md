# 0x492442eab586f242b53bda933fd5de859c8a3782 트레이더 프로필

## 요약

<!-- para: para_001 -->
> 2026-04-05 기준 월간 리더보드 1위. $848k PnL, $823k 거래량.

<!-- para: para_002 -->
이 지갑 주소는 Polymarket 월간 리더보드에서 1위를 기록한 계정이다(2026-04-05 스냅샷). 현재까지 공개된 소셜 핸들(트위터/X 등)이 확인되지 않아, 온체인 행동 패턴과 포지션 히스토리로만 분석해야 하는 유형의 대표적인 사례이다.

<!-- para: para_003 -->
| 지표 | 값 |
|------|------|
| 리더보드 순위(월간) | 1위 |
| 월간 PnL | $848,046 |
| 월간 거래량 | $823,673 |
| PnL 대비 거래량 비율 | 0.97 (높은 자본 효율) |

PnL/거래량 비율이 0.97에 가까운 것은 이 트레이더가 **높은 승률 + 효율적 자본 회전**을 보여준다는 신호다. 일반적으로 이 비율이 1에 가까울수록 거래량 대비 순이익이 크다는 의미이나, 단일 스냅샷이므로 기간별 추이를 확인해야 한다.

<!-- para: para_004 -->
이 지갑의 PnL($848k)이 거래량($823k)과 거의 같은 것은 다음과 같은 해석이 가능하다:

1. **높은 승률형**: 대부분의 포지션이 수익으로 마감
2. **회전율 제한**: 동일 자본을 여러 번 회전하지 않고 집중 베팅

<!-- para: para_005 -->
| 기준 | 평가 |
|------|------|
| 초기 PnL | 우수 — $848k 월간 |
| 자본 효율 | 높음 — PnL/Volume ≈ 0.97 |
| 샘플 크기 | 부족 — 단일 스냅샷 |
| 전략 투명성 | 낮음 — 공개 소셜 부재 |
| 검증 상태 | 미검증 — 장기 데이터 확보 필요 |

<!-- para: para_006 -->
1. **3개월 관찰**: 최소 3개월간 월간 순위 유지 여부 확인 ([[리더보드 & 트레이더 데이터]])
2. **포지션 히스토리 분석**: Gamma API로 최근 진입/청산 패턴 수집

<!-- para: para_007 -->
- **장점**: 높은 자본 효율, 대형 마켓 중심 가능성 → 슬리피지 작음
- **위험**: 단일 달 성과로 판단할 수 없음, 전략 변경 가능성 높음
- **권장**: watch-list에 올리고 3개월 후 [[Copytrade 데이터 수집 가이드]] 기준 재평가
- **만약 팔로우 시**: 계정 자본의 5-10%만 할당, 유동성 < $1K 마켓 스킵

## 세부 내용

<!-- para: para_008 -->
Polymarket의 리더보드 페이지로, 오늘/주간/월간/전체 탭과 카테고리 필터를 통해 순위를 보여준다. 제공된 스냅샷은 월간, 전체 카테고리, `Profit/Loss` 기준이며 상위 20명의 사용자명, 손익, 거래량이 표시된다. 아래에는 이번 달 큰 수익 항목을 모은 섹션이 있어 사용자명과 관련 시장명이 함께 나온다. 하단에는 Polymarket 관련 카테고리 링크와 `Terms of Service`, `Privacy Policy`, `Market Integrity`를 포함한 규제·위험 고지가 있다.

<!-- para: para_009 -->
Polymarket의 `GET /v1/leaderboard` 엔드포인트는 `https://data-api.polymarket.com/v1/leaderboard`에서 트레이더 순위를 조회할 때 사용합니다. 응답은 `rank`, `proxyWallet`, `userName`, `vol`, `pnl`, `profileImage`, `xUsername`, `verifiedBadge` 필드를 포함한 JSON 배열로 반환됩니다. 사용할 수 있는 쿼리 매개변수는 `category`(OVERALL, POLITICS, SPORTS, CRYPTO, CULTURE, MENTIONS, WEATHER, ECONOMICS, TECH, FINANCE), `timePeriod`(DAY, WEEK, MONTH, ALL), `orderBy`(PNL, VOL), `limit`(1~50, 기본값 25), `offset`(0~1000, 기본값 0), `user`, `userName`입니다.

<!-- para: para_010 -->
Polymarket Leaderboard는 Polymarket Data API를 바탕으로 만든 리더보드형 화면으로, 상위 트레이더와 큰 수익을 한눈에 보여줍니다. `Overall`/`Politics`/`Sports`/`Crypto` 같은 카테고리와 `1 Day`/`7 Days`/`1 Month`/`All Time` 기간 필터, `Watchlist`, `Export CSV` 기능이 있으며, `PNL`과 `Volume` 정렬은 클라이언트에서 처리합니다. `Smart Money` 분류는 코드에 정의된 성과 기준과 선택적으로 적용되는 `WalletLabels` 신호를 함께 사용합니다.

<!-- para: para_011 -->
이 문서는 Polymarket Data SDK의 Gamma API에서 `client.gamma.search.publicSearch(params)`를 호출하는 방법을 설명합니다. `q`를 지정하면 `events`, `tags`, `profiles`를 대상으로 전체 텍스트 검색을 수행할 수 있고, 필요에 따라 `events_status`, `events_tag`, `search_tags`, `search_profiles`, `limit_per_type`, `page`, `sort`, `ascending`, `recurrence`, `exclude_tag_id`, `keep_closed_markets`, `cache`, `optimized` 같은 필터와 옵션도 함께 설정할 수 있습니다. 반환값에는 `events`, `tags`, `profiles`, `pagination { hasMore, totalResults }`가 포함됩니다. 각 항목은 `null` 또는 `undefined`일 수 있으므로, 실제 코드에서는 존재 여부를 확인하면서 처리해야 합니다. 문서에는 요청 형식, 매개변수, 응답 구조, 사용 예시, 실패 사례가 순서대로 정리되어 있습니다.
