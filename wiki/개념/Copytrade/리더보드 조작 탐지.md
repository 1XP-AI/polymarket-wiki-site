# 리더보드 조작 탐지

## 요약

<!-- para: para_001 -->
> 한 줄 요약: 리더보드 상의 비정상적 패턴을 식별하여 copytrade 후보의 신뢰도를 평가하는 방법과 지표를 정리한다.

<!-- para: para_002 -->
리더보드는 copytrade의 핵심 출발점이다. 그러나 리더보드에 등재된 트레이더 중 일부는 인위적 활동(계정 군집, 자전거래, 봇 활동 등)으로 인해 과대평가될 수 있다.

<!-- para: para_003 -->
크게 4개 범주로 분류한다.

<!-- para: para_004 -->
| 지표 | 계산법 | 임계값 | 패턴 설명 |
|------|--------|--------|----------|
| 시간 주기성 | FFT 또는 자기상관 분석 | autocorr > 0.7 at fixed lag | 매 15분/1시간마다 자동 매매 |
| 베팅 크기 변동성 | std(포지션크기) / mean(포지션크기) | CV < 0.1 | 봇 고정 금액 패턴 |
| 승률-거래량 괴리 | 승률 × 거래횟수 대비 총 PnL | high win_rate, low volume | 소액 다수로 승률 조작 |
| 주문-취소 비율 | canceled / (filled + canceled) | ratio > 0.8 | 스푸핑(spoofing) 의심 |
| 진입-청산 간격 | mean(청산시간 - 진입시간) | < 30초 반복 | 마켓 메이킹/아비트리지 봇 |

<!-- para: para_005 -->
- **동시 행동 군집**: 서로 다른 지갑이 ±5초 내 동일 마켓 동일 방향 진입
- **자금 순환**: 지갑 A → B → C → A로 USDC 순환 (wash trading)
- **공급자 동일성**: 여러 계정이 동일한 funding source(동일 CEX 출금 주소) 사용
- **신생 지갑 활동**: 생성 후 48시간 내 리더보드 진입 → 대량 거래 시작

<!-- para: para_006 -->
- **자기 거래(self-trading)**: 자신이 만든 유동성 풀에서 자신과 거래
  - 탐지: 지갑 A의 buy와 지갑 B의 sell이 동일 이벤트, 동일 가격, 근접 시간
- **마스터-슬레이브 구조**: 메인 계정으로 시장 형성 후 서브 계정으로 추승
- **초과 수익의 통계적 검증**: Sharpe ratio > 3 지속 → 무작위성 검정 실패 의심

```python
def detect_wash_trading(trades, window_seconds=60):
    """
    자기 거래 탐지: 동일 마켓에서 반대 방향, 근접 시간, 유사 금액
    """
    suspicious_pairs = []
    for i, t1 in enumerate(trades):
        for t2 in trades[i+1:]:
            if (t2['timestamp'] - t1['timestamp']).total_seconds() > window_seconds:
                break
            if (t1['market_id'] == t2['market_id'] and
                t1['side'] != t2['side'] and
                abs(t1['size'] - t2['size']) / t1['size'] < 0.05 and
                t1['wallet'] != t2['wallet']):  # 다른 지갑이지만
                suspicious_pairs.append((t1, t2))
    return suspicious_pairs
```

<!-- para: para_007 -->
- 트위터/디스코드에서 주장하는 포지션과 실제 온체인 포지션 불일치
- "대박 수익" 캡처 vs 실제 지갑 PnL 차이
- 팔로워 유도용 가짜 신호 발송 (pump-and-dump variant)
