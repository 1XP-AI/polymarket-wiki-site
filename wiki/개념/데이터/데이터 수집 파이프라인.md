# 데이터 수집 파이프라인

## 요약

<!-- para: para_001 -->
> raw 소스에서 위키용 마크다운까지의 표준 파이프라인 — Copytrade 전략에 필수적인 트레이더·마켓·포지션 데이터의 수집·저장·검증 아키텍처

<!-- para: para_002 -->
데이터 수집 파이프라인은 소스 발견 → 수집 → 변환 → 저장 → 인덱싱의 5단계로 구성된다. Polymarket Copytrade 컨텍스트에서는 **트레이더 행동 데이터**(리더보드, 포지션, 거래내역)와 **마켓 컨텍스트**(가격, 유동성, 이벤트)가 핵심 수집 대상이다.

<!-- para: para_003 -->
| API | 엔드포인트 | 용도 | 수집 주기 |
|-----|-----------|------|----------|
| Gamma API | `/events`, `/markets`, `/profiles` | 마켓 목록, 이벤트 메타, 공개 프로필 | 6시간 |
| Data API | `/v1/leaderboard`, `/positions`, `/trades` | 리더보드 순위, 트레이더 포지션/거래 | 1시간 |
| CLOB API | `/rewards/leaderboard`, `/orderbook` | 오더북, 리베이트 리더보드 | 30분 |
| WebSocket | `wss://ws.polymarket.com` | 실시간 체결, 오더북 업데이트 | 지속 |

<!-- para: para_004 -->
curl -s "https://gamma-api.polymarket.com/events?limit=20&active=true"

<!-- para: para_005 -->
curl -s "https://data-api.polymarket.com/v1/leaderboard?window=monthly&limit=50"

<!-- para: para_006 -->
curl -s "https://data-api.polymarket.com/positions?address=0x1234..."
