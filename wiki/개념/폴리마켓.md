# 폴리마켓 개요

> 한 줄 요약: 폴리마켓은 Polygon 기반의 주요 예측시장 플랫폼으로, 이벤트/마켓/오더북 기반 거래를 제공한다.

## 내용

폴리마켓은 사용자들이 실제 사건의 결과에 베팅하는 예측시장 플랫폼이다. 주요 구성 요소는 이벤트(Event), 마켓(Market), 오더북(중앙지정가주문장, CLOB)이며 가격은 0~1 사이로 확률을 나타낸다. 마켓과 이벤트 정보는 Gamma API를 통해 조회할 수 있고, 포지션은 Yes/No 토큰 형태로 발행되어 결과에 따라 정산된다.

### 핵심 구성요소 요약

- 이벤트(Event): 실제 세계의 사건 단위(예: 선거 결과, 스포츠 경기).
- 마켓(Market): 특정 이벤트에 대한 베팅 창구(예: "후보 A 승리 여부").
- 주문장(CLOB): 가격·수량 기반의 주문 매칭 장부. 대규모 트레이더의 행동을 이해하는 데 중요.

## 예제: Gamma API 호출 (간단한 조회)

아래 예제는 공개적으로 접근 가능한 Gamma API를 사용한 간단한 마켓 목록 조회 예시이다.

Bash (curl):

```bash
# 최근 활성 마켓 5개 조회
curl -s "https://gamma-api.polymarket.com/markets?limit=5" | jq '.'
```

Python (requests):

```python
import requests

url = "https://gamma-api.polymarket.com/markets?limit=5"
resp = requests.get(url, timeout=5)
if resp.status_code == 200:
    data = resp.json()
    for m in data.get('markets', [])[:5]:
        print(m.get('id'), m.get('name'))
else:
    print('HTTP', resp.status_code)
```

위 예제는 공개 API 호출 샘플로, 실제 자동화 파이프라인에서는 rate limit과 에러 처리를 반드시 포함해야 한다.

데이터 파이프라인 팁:

- 권장 엔드포인트: Gamma markets/events, leaderboard 및 Polymarket의 공식 문서 페이지를 정기 스냅샷하라. 예: https://gamma-api.polymarket.com/markets, https://gamma-api.polymarket.com/events, https://data.polymarket.com/leaderboard
- 스냅샷 주기: 활동 마켓은 분 단위(또는 WebSocket), 일반 마켓은 5-30분 주기 권장
- 저장 형식: raw/fetched/YYYY-MM-DD-주제.md 형식으로 메타(타임스탬프, 응답 원문)와 함께 보관
- 운영 고려: rate limit, 재시도 정책, 타임아웃 및 타임스탬프 동기화(UTC 권장)를 반드시 구현하여 백테스트와 실행 로그의 정합성을 보장한다.

## 마켓 유형

### CLOB 기반 마켓
가격·수량 기반 중앙지정가주문장(CLOB) 매칭으로, 지정가 주문을 제출해 유동성을 제공하거나 시장가 주문으로 즉시 체결할 수 있습니다. 고빈도 트레이더의 활동이 활발하며, 슬리피지가 낮습니다(관련: [[개념/CLOB]]).

### Neg-Risk 마켓
다중 결과 이벤트(예: 대선에서 3명 이상의 후보)에서 독립적인 Yes/No 마켓을 병렬로 운용합니다. CTF의 Split/Merge 없이도 유사한 구조를 제공하며, `negRisk: true` 플래그가 필요합니다. Neg-Risk 거래량은 $14B+로 CTF를 추월 중입니다(관련: [[개념/Negative-Risk]]).

### CTF 기반 마켓
조건부 토큰을 Split/Merge하여 포지션을 구성합니다. 모든 결과가 서로 연결되므로 Capital Efficiency가 높지만, 토큰 조합 복잡도가 있습니다(관련: [[개념/CTF]]).

## 정산과 리스크

### UMA 오라클 정산
결과는 UMA Optimistic Oracle에 의해 확정됩니다. 결과 제출 후에는 도전 기간이 있으며, 최종 확정 후 승리 토큰을 Redeem하여 $1.00을 수령합니다.

### 주요 리스크
- **API rate limit**: 과도한 호출 시 429 응답. exponential backoff 권장
- **오라클 지연**: 결과 확정 지연 시 포지션 청산 불가
- **슬리피지**: 얕은 오더북에서 대형 주문은 가격에 큰 영향 (관련: [[개념/슬리피지]])

## Copytrade 시사점
- 데이터 파이프라인의 스냅샷 주기에 따라 신호 탐지 속도가 결정됨
- Neg-Risk 마켓은 초보 카피 트레이더에게 진입 장벽이 낮음
- CLOB 시장에서는 Maker 리베이트를 활용하면 비용 절감 가능([[전략/마켓메이커/리베이트]])

## 실행/리스크 고려사항

- API 응답은 시시각각 변하므로 스냅샷 저장(raw/fetched/)을 권장.
- 주문장(CLOB) 관찰 시 대형 주문(whale)의 임팩트를 고려해 슬리피지(slippage) 및 시장충격을 모델링해야 함.
- copytrade 목적으로 트레이더를 추종할 때는 거래 속도와 주문 실행 지연(delay)을 반드시 테스트.
