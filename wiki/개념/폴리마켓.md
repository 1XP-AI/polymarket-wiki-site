# 폴리마켓 개요

> 한 줄 요약: 폴리마켓은 Polygon 기반의 주요 예측시장 플랫폼으로, 이벤트/마켓/오더북 기반 거래를 제공한다.

## 내용

폴리마켓은 사용자들이 실제 사건의 결과에 베팅하는 예측시장 플랫폼이다. 주요 구성 요소는 이벤트(Event), 마켓(Market), 오더북(중앙지정가주문장, CLOB)이며 가격은 0~1 사이로 확률을 나타낸다. 마켓과 이벤트 정보는 Gamma API를 통해 조회할 수 있고, 포지션은 Yes/No 토큰 형태로 발행되어 결과에 따라 정산된다.

### 핵심 구성요소 요약

- 이벤트(Event): 실제 세계의 사건 단위(예: 선거 결과, 스포츠 경기).
- 마켓(Market): 특정 이벤트에 대한 베팅 창구(예: "후보 A 승리 여부").
- 주문장(CLOB): 가격·수량 기반의 주문 매칭 장부. 대규모 트레이더의 행동을 이해하는 데 중요.

## 예제: Gamma API 호출 (간단한 조회)

아래 예제는 공개적으로 접근 가능한 Gamma API를 사용한 간단한 마켓 목록 조회 예시이다.

Bash (curl):

```bash
# 최근 활성 마켓 5개 조회
curl -s "https://gamma-api.polymarket.com/markets?limit=5" | jq '.'
```

Python (requests):

```python
import requests

url = "https://gamma-api.polymarket.com/markets?limit=5"
resp = requests.get(url, timeout=5)
if resp.status_code == 200:
    data = resp.json()
    for m in data.get('markets', [])[:5]:
        print(m.get('id'), m.get('name'))
else:
    print('HTTP', resp.status_code)
```

위 예제는 공개 API 호출 샘플로, 실제 자동화 파이프라인에서는 rate limit과 에러 처리를 반드시 포함해야 한다.

## 실행/리스크 고려사항

- API 응답은 시시각각 변하므로 스냅샷 저장(raw/fetched/)을 권장.
- 주문장(CLOB) 관찰 시 대형 주문(whale)의 임팩트를 고려해 슬리피지(slippage) 및 시장충격을 모델링해야 함.
- copytrade 목적으로 트레이더를 추종할 때는 거래 속도와 주문 실행 지연(delay)을 반드시 테스트.
