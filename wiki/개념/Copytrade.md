# Copytrade

> 리더보드 트레이더를 따라 거래하는 전략 설명

## 내용

Copytrade는 리더보드 상의 검증된 트레이더를 추종하여 그들의 포지션 변화를 자동으로 복제하는 전략이다. 목적은 숙련된 트레이더이 가진 통찰(알파)을 활용하면서, 개별 의사결정의 로컬 최적화(심리적 오류 등)를 회피하는 것이다. 본 문서는 Copytrade의 핵심 개념, 선택 기준, 리스크 관리 원칙 및 구현 고려사항을 요약한다.

### 핵심 아이디어

- 리더보드에 반복적으로 높은 성과를 내는 트레이더의 행동 패턴(진입·청산 타이밍, 선호 마켓, 포지션 규모)을 식별한다.
- 식별된 패턴을 규칙 또는 모델로 구현하여 자동으로 추종한다.
- 단순 무작정 복제는 큰 리스크를 초래하므로 포지션 사이즈 조정, 리밸런싱, 손실 제한 규칙을 반드시 포함한다.

### 선택 기준 (예시)

1. 일관성: 최근 30~90일간의 승률 및 수익의 일관성
2. 샘플 크기: 포지션 빈도/거래 횟수가 충분한지(과적합 방지)
3. 익스포저 유형: 단기 베팅인지, 사건 기반 장기 베팅인지 구분
4. 시장 적합성: 해당 트레이더의 전략이 우리가 복제하려는 시장(크립토/스포츠/정책 등)에 적합한지
5. 온체인·오프체인 근거: 프로필·소셜·과거 성과로 신뢰성 검토

### 리스크 관리(필수)

- 포지션 사이즈 한도: 계정 자본의 일정 비율 상한(예: 1~3%)을 설정
- 손실 컷오프: 누적 손실이 일정 수준(예: 전체 자본의 5%)을 넘으면 추종 중단
- 중복 노출 제한: 동일한 이벤트·주제에 과도하게 노출되지 않도록 클러스터링 검사
- 추종 적중 실패 시 플래그: 트레이더 성과가 갑자기 악화되면 자동으로 팔로우 중단
- 슬리피지·수수료 고려: 거래 비용을 반영한 기대수익 계산

### 구현 고려사항

- 데이터 소스: 리더보드 스냅샷(raw/fetched/), Polymarket API([[API-레퍼런스]])에서 마켓·포지션 데이터 수집
- 지연(레이턴시): 트레이더의 포지션은 실시간에 가깝지만 복제 실행에는 지연이 발생할 수 있으므로, 추종 속도 최적화가 필요하다(추종-실행-속도-최적화 참조)
- 자동화 로직: 단순 규칙 기반(포지션 감지 → 동일 비율 진입) 또는 모델 기반(확률적 가중치 적용) 선택
- 시뮬레이션: 백테스트(과거 리더보드·마켓 스냅샷 사용)와 샌드박스 환경에서의 시뮬레이션을 우선 수행

### 간단한 예시 (의사코드)

- 트레이더 A가 시장 X에 'Yes' 포지션을 구축하는 것을 감지하면:
  1) 해당 마켓의 유동성·임박성·수수료 계산
  2) 사전 정의된 리스크 규칙(최대 비율, 중복 노출 검사) 통과 시 추종
  3) 실행 후 모니터링: 포지션 변경 시 재조정

> 구현 시 실제 API 호출 예시는 [[API-레퍼런스]]와 위키의 데이터 수집 가이드를 참고한다.
