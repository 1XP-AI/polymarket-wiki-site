# 네거티브 리스크

## 요약

<!-- para: para_001 -->
> 다중 결과 이벤트에서 자본 효율적으로 포지션을 구성해 손실을 제한하거나 구조적으로 차익 기회를 만드는 접근이다.

<!-- para: para_002 -->
네거티브 리스크는 여러 결과가 연결된 이벤트에서 특정 결과를 매수하거나 나머지 결과를 조합해, 어떤 결과가 나와도 손익 구조를 유리하게 만드는 전략이다. 실무에서는 헤지와 차익 거래 아이디어가 함께 섞여 쓰인다.

<!-- para: para_003 -->
다중 결과 마켓에서는 하나의 이벤트를 여러 결과 토큰으로 나눠 생각할 수 있다. 이 구조 덕분에 특정 결과를 직접 매수하는 방식뿐 아니라, 나머지 결과를 반대 포지션으로 조합해 사실상 동일한 경제적 노출을 만들 수 있다.

<!-- para: para_004 -->
세 후보 A, B, C가 있고 각각 시장 가격이 $0.40, $0.35, $0.30이라고 가정합시다.

| 전략 | 비용 | A 당첨 시 | B 당첨 시 | C 당첨 시 |
|------|------|----------|----------|----------|
| Yes-A 매수 | $0.40 | $1.00 | $0.00 | $0.00 |
| Yes-B 매수 | $0.35 | $0.00 | $1.00 | $0.00 |
| Yes-C 매수 | $0.30 | $0.00 | $0.00 | $1.00 |
| **전부 매수** | **$1.05** | **$1.00** | **$1.00** | **$1.00** |

이 경우 세 결과를 모두 사면 $0.05의 확정 손실이다. 반대로 가격 합이 $1.00보다 낮아지는 순간에는 모든 Yes를 묶어 사는 것만으로 확정 수익 기회가 생긴다.

<!-- para: para_005 -->
실무에서는 모든 Yes 가격의 합이 $1.00보다 큰지, 작은지를 먼저 본다. 합이 $1.00보다 크면 과대평가된 구조이고, $1.00보다 작으면 저평가된 구조다. 뉴스 직후처럼 시장이 급하게 움직일 때 잠깐 이런 불균형이 생길 수 있다.

<!-- para: para_006 -->
Copytrade 문맥에서는 네거티브 리스크를 공격용 전략보다 헤지 수단으로 보는 편이 실무적이다. 특정 트레이더가 정치 마켓 한쪽 방향으로 크게 쏠렸을 때, 나머지 결과를 조합해 하방을 제한하는 방식으로 쓸 수 있다.

<!-- para: para_007 -->
1. **단일 베팅 헤지**: 특정 정치 마켓에서 Yes에 진입한 후, 나머지 모든 No를 합성해 구매해 손실 한도를 고정한다.
2. **카테고리 분산**: 정치 마켓 전문 트레이더를 카피할 때, 해당 트레이더가 집중하는 마켓의 반대 포지션을 추가해 downside를 제한한다.

<!-- para: para_008 -->
Neg-Risk 마켓과 일반 다중 결과 마켓은 비슷해 보여도 실행 방식이 다르다. 전자는 독립 Yes/No 마켓을 묶어 다루는 경향이 강하고, 후자는 구조적으로 결과 토큰이 연결되어 있어 Split/Merge 사고방식으로 이해하는 편이 쉽다.

| 구분 | Neg-Risk 마켓 | 일반 다중 결과 마켓 |
|------|---------------|----------------------|
| 구조 | 독립 Yes/No 마켓을 조합 | 결과 토큰이 구조적으로 연결 |
| 핵심 활용 | 헤지, 차익 기회 탐지 | 결과별 확률 거래 |
| 실무 난점 | 동시 실행과 슬리피지 관리 | 토큰 조합과 정산 구조 이해 |

## 구현 예시

<!-- para: para_009 -->
```python
def check_negative_risk_opportunity(market_prices):
    """모든 Yes 가격 합을 기준으로 Neg-Risk 기회를 점검한다."""
    total_yes = sum(item["yes_price"] for item in market_prices)

    if total_yes < 1.0:
        return {
            "action": "buy_all_yes",
            "guaranteed_profit": round(1.0 - total_yes, 4),
        }

    if total_yes > 1.0:
        total_no = sum(1 - item["yes_price"] for item in market_prices)
        return {
            "action": "sell_all_no",
            "guaranteed_profit": round(total_no - 1.0, 4),
        }

    return None
```

<!-- para: para_010 -->
- 슬리피지와 수수료 때문에 이론상 차익이 실제 손익으로 바로 이어지지 않을 수 있다.
- 유동성이 얕은 마켓은 조합 주문을 맞추는 도중 가격이 먼저 움직일 수 있다.
- 구조는 매력적이지만, 실제 운용에서는 실행 속도와 체결 품질이 성패를 좌우한다.
