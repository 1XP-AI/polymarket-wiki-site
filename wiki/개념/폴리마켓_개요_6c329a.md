# 폴리마켓 개요

## 요약

<!-- para: para_001 -->
> 한 줄 요약: 폴리마켓은 Polygon 기반의 주요 예측시장 플랫폼으로, 이벤트/마켓/오더북 기반 거래를 제공한다.

## 핵심 사실

<!-- para: para_002 -->
폴리마켓은 사용자들이 실제 사건의 결과에 베팅하는 예측시장 플랫폼이다. 주요 구성 요소는 이벤트(Event), 마켓(Market), 오더북(중앙지정가주문장, CLOB)이며 가격은 0~1 사이로 확률을 나타낸다.

<!-- para: para_003 -->
- 이벤트(Event): 실제 세계의 사건 단위(예: 선거 결과, 스포츠 경기). - 마켓(Market): 특정 이벤트에 대한 베팅 창구(예: "후보 A 승리 여부").

## 세부 내용

<!-- para: para_004 -->
아래 예제는 공개적으로 접근 가능한 Gamma API를 사용한 간단한 마켓 목록 조회 예시이다. Bash (curl):

```bash

<!-- para: para_005 -->
curl -s "https://gamma-api.polymarket.com/markets?limit=5" | jq '.'
```

Python (requests):

```python
import requests

url = "https://gamma-api.polymarket.com/markets?limit=5"
resp = requests.get(url, timeout=5)
if resp.status_code == 200:
    data = resp.json()
    for m in data.get('markets', [])[:5]:
        print(m.get('id'), m.get('name'))
else:
    print('HTTP', resp.status_code)
```

위 예제는 공개 API 호출 샘플로, 실제 자동화 파이프라인에서는 rate limit과 에러 처리를 반드시 포함해야 한다. 데이터 파이프라인 팁:

- 권장 엔드포인트: Gamma markets/events, leaderboard 및 Polymarket의 공식 문서 페이지를 정기 스냅샷하라. 예: https://gamma-api.polymarket.com/markets, https://gamma-api.polymarket.com/events, https://data.polymarket.com/leaderboard
- 스냅샷 주기: 활동 마켓은 분 단위(또는 WebSocket), 일반 마켓은 5-30분 주기 권장
- 저장 형식: raw/fetched/YYYY-MM-DD-주제.md 형식으로 메타(타임스탬프, 응답 원문)와 함께 보관
- 운영 고려: rate limit, 재시도 정책, 타임아웃 및 타임스탬프 동기화(UTC 권장)를 반드시 구현하여 백테스트와 실행 로그의 정합성을 보장한다.

<!-- para: para_006 -->
가격·수량 기반 중앙지정가주문장(CLOB) 매칭으로, 지정가 주문을 제출해 유동성을 제공하거나 시장가 주문으로 즉시 체결할 수 있습니다. 고빈도 트레이더의 활동이 활발하며, 슬리피지가 낮습니다(관련: [[개념/CLOB]]).

<!-- para: para_007 -->
다중 결과 이벤트(예: 대선에서 3명 이상의 후보)에서 독립적인 Yes/No 마켓을 병렬로 운용합니다. CTF의 Split/Merge 없이도 유사한 구조를 제공하며, `negRisk: true` 플래그가 필요합니다. Neg-Risk 거래량은 $14B+로 CTF를 추월 중입니다(관련: [[개념/Negative-Risk]]).
