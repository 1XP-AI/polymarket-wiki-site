# 온체인 컨트랙트

## 요약

<!-- para: para_001 -->
> 한 줄 요약: Polymarket이 Polygon 체인에서 사용하는 온체인 컴포넌트(CTF 토큰 발행·스플릿/머지·정산, NegRiskExchange, Proxy/Safe)의 구조와 Copytrade에서의 온체인 데이터 활용법을 설명합니다.

<!-- para: para_002 -->
Polymarket의 핵심 로직은 **Polygon PoS** (Chain ID: 137) 위의 스마트컨트랙트에서 실행된다. 오프체인 CLOB에서 주문 매칭이 이루어지지만, 실제 토큰 발행·정산·유저 자산 관리는 온체인 컨트랙트가 담당한다.

<!-- para: para_003 -->
사용자 브라우저 ─→ Polymarket 프론트엔드
                      │
                      ├─→ Gamma API (Gamma): 이벤트/마켓/오더북 읽기
                      ├─→ CLOB API (clob.polymarket.com): 주문 작성/취소
                      │
                      └─→ Polygon PoS (Chain 137): 온체인 정산
                              │
                              ├─ CTF 토큰 컨트랙트
                              ├─ NegRiskExchange 컨트랙트
                              └─ Proxy/Safe 컨트랙트

<!-- para: para_004 -->
- 낮은 가스비: 수백~수천 건의 소규모 거래에 적합 (평균 $0.001 미만)
- EVM 호환성: 기존의 이더리움 도구(MetaMask, Hardhat, ethers.js)를 그대로 사용
- 빠른 블록 타임: ~2초 블록으로 실시간 거래 경험 제공
- Polymarket의 모든 정산과 토큰 발행은 이 체인 위에서 최종적으로 확정된다

<!-- para: para_005 -->
Gnosis(현 Conditional Tokens)에서 개발한 ERC-1155 기반 스마트 컨트랙트 표준. Polymarket 초기 핵심 인프라로, 하나의 예측 이벤트를 여러 조건부 결과 토큰으로 분할하고, 거래 후 다시 병합하여 정산할 수 있게 한다. **핵심 불변식: `1 USDC = Yes₁ + Yes₂ + … + Yesₙ`** (n개 결과의 완전한 집합)

$1을 컨트랙트에 입금(split)하면 모든 결과 토큰을 각각 1개 받고, 모든 결과 토큰을 1개씩 모아 컨트랙트에 보내면(join/merge) $1을 돌려받는다.

<!-- para: para_006 -->
각 position ID는 `(collateralToken, conditionId, indexSet)`의 해시로 uniquely 결정된다. Copytrade 봇은 이 토큰 ID를 해석해서 "어떤 마켓의 어떤 결과에 베팅했는지"를 복원해야 한다.

<!-- para: para_007 -->
[[개념/Negative-Risk]] 마켓 전용. 3개 이상의 결과가 있는 마켓에서 자본 효율적인 포지션 구성을 지원한다. **일반 Yes/No vs Negative-Risk 비교:**

| 항목 | 일반 (Yes/No) | Negative-Risk |
|------|---------------|---------------|
| 결과 수 | 2 | 3개 이상 |
| Split | collateral → Yes + No | collateral → A + B + C + ...
