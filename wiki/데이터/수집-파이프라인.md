# 데이터 수집 파이프라인

> raw 소스에서 위키용 마크다운까지의 표준 파이프라인 — Copytrade 전략에 필수적인 트레이더·마켓·포지션 데이터의 수집·저장·검증 아키텍처

## 파이프라인 아키텍처

데이터 수집 파이프라인은 소스 발견 → 수집 → 변환 → 저장 → 인덱싱의 5단계로 구성된다. Polymarket Copytrade 컨텍스트에서는 **트레이더 행동 데이터**(리더보드, 포지션, 거래내역)와 **마켓 컨텍스트**(가격, 유동성, 이벤트)가 핵심 수집 대상이다.

## 수집 소스 분류

### 1. 공식 API (우선순위 1)

| API | 엔드포인트 | 용도 | 수집 주기 |
|-----|-----------|------|----------|
| Gamma API | `/events`, `/markets`, `/profiles` | 마켓 목록, 이벤트 메타, 공개 프로필 | 6시간 |
| Data API | `/v1/leaderboard`, `/positions`, `/trades` | 리더보드 순위, 트레이더 포지션/거래 | 1시간 |
| CLOB API | `/rewards/leaderboard`, `/orderbook` | 오더북, 리베이트 리더보드 | 30분 |
| WebSocket | `wss://ws.polymarket.com` | 실시간 체결, 오더북 업데이트 | 지속 |

```bash
# Gamma API — 활성 이벤트
curl -s "https://gamma-api.polymarket.com/events?limit=20&active=true"

# Data API — 리더보드
curl -s "https://data-api.polymarket.com/v1/leaderboard?window=monthly&limit=50"

# Data API — 특정 트레이더 포지션
curl -s "https://data-api.polymarket.com/positions?address=0x1234..."
```

### 2. 외부 소스 (우선순위 2)

- **GitHub**: 오픈소스 copytrade 봇 구현체 참고 (`agc-desarrollo/polymarket-trading-bot`, `Fabianhvandijk/polymarket-copytrade`)
- **분석 플랫폼**: `polymarketanalytics.com/traders` — 트레이더 통계 대시보드
- **커뮤니티**: Reddit `r/polymarket`, Twitter `@Polymarket` — 신규 마켓/이벤트 조기 감지

## 단계별 상세 절차

### 1. 소스 확인

`raw/sources.md`를 읽어 수집 대상을 선정한다. 신규 소스 발견 시 즉시 등록하고, 기존 소스는 마지막 수집 시각을 기록하여 중복을 방지한다.

### 2. 수집(Fetch)

각 소스별로 적절한 도구를 사용:

```bash
# DuckDuckGo 검색 (외부 참조용)
ddgr "polymarket copytrade leaderboard" --json -n 10

# Polymarket API (페이징 처리)
curl -s "https://gamma-api.polymarket.com/markets?limit=50&offset=0" > page1.json
curl -s "https://gamma-api.polymarket.com/markets?limit=50&offset=50" > page2.json
```

**수집 실패 처리**:
- HTTP 429 (Rate Limit): 지수 백오프 (1s → 2s → 4s → 8s → 최대 30s)
- HTTP 5xx: 3회 재시도 후 다음 소스로 전환, 원인 로그 기록
- 타임아웃: 기본 10초, 중요 API는 30초로 상향

### 3. 변환(Transform)

JSON/HTML을 위키용 마크다운으로 변환:

```python
import json, yaml

def transform_to_wiki(raw_data: dict, source: str) -> str:
    """원본 JSON을 위키 마크다운으로 변환"""
    frontmatter = {
        "title": "수집 데이터 스냅샷",
        "source": source,
        "fetched": "2026-04-07",
        "endpoint": "/markets"
    }
    header = yaml.dump(frontmatter, default_flow_style=False)
    body = "```json\n" + json.dumps(raw_data, indent=2) + "\n```\n"
    return f"---\n{header}---\n\n{body}"
```

### 4. 저장(Store)

- 파일명 규칙: `raw/fetched/YYYY-MM-DD-소스-주제.md`
- 원본 보존: `raw/fetched/original/YYYY-MM-DD-source.json`
- 동일 소스 반복 수집 시 최신 파일이 이전 파일을 대체하지 않고 별도 저장

### 5. 인덱싱(Index)

수집된 데이터에서 위키에 유용한 항목 추출:

1. **트레이더**: 지갑 주소, 핸들, 리더보드 순위
2. **마켓**: market_id, 이벤트명, 카테고리, 현재 가격
3. **신호**: 신규 대형 포지션, 급변하는 마켓 동향

추출 결과는 `raw/sources.md`에 누적하고 `_fetch-log.md`에 수집 메타데이터 기록.

### 6. 적합성 평가 (Relevance Gate)

wiki-writer.md의 평가 기준에 따라 저장 여부 결정:

| 기준 | 통과 조건 |
|------|----------|
| 직접 관련 | Polymarket 플랫폼/트레이딩 직접 언급 |
| 전략 활용 | Copytrade 파이프라인에 활용 가능한 데이터 패턴 |
| 신규 인사이트 | 기존 문서가 다루지 않은 트레이더 행동/마켓 구조 |

## 파이프라인 자동화 예시

```python
import requests, time, json
from datetime import datetime

class PolymarketCollector:
    BASE = "https://gamma-api.polymarket.com"
    DATA = "https://data-api.polymarket.com/v1"
    
    def __init__(self, delay=60):
        self.session = requests.Session()
        self.delay = delay  # 수집 간격(초)
    
    def collect_leaderboard(self, window="monthly", limit=50):
        resp = self.session.get(f"{self.DATA}/leaderboard", 
            params={"window": window, "limit": limit})
        if resp.ok:
            data = resp.json()
            filename = f"raw/fetched/{datetime.now():%Y-%m-%d}-leaderboard-{window}.md"
            with open(filename, "w") as f:
                f.write(f"# Leaderboard Snapshot: {window}\n")
                f.write(f"```json\n{json.dumps(data, indent=2)}\n```\n")
            return data
        return None
    
    def collect_market(self, limit=50):
        resp = self.session.get(f"{self.BASE}/markets", 
            params={"limit": limit})
        if resp.ok:
            return resp.json()
        return None
    
    def run_cycle(self):
        """단일 수집 사이클 실행"""
        lb = self.collect_leaderboard()
        markets = self.collect_market()
        time.sleep(self.delay)
        return {"leaderboard": lb, "markets": markets}
```

## 운영 패턴

### Copytrade용 수집 우선순위

1. **실시간 감지 단계** (`~5분`): WebSocket + REST 폴링으로 리더 변동/대형 포지션 감지
2. **단기 분석 단계** (`1~6시간`): 포지션 스냅샷, 거래내역 수집으로 패턴 분석
3. **장기 보관 단계** (`일간/주간`): 리더보드 전체 스냅샷, 마켓 카탈로그 업데이트

### Rate Limit 주의사항

- Data API: 초당 10회 권장, 초과 시 429 응답
- Gamma API: 비교적 관대하지만 100회/분 이하 권장
- `backoff_factor=1`로 HTTPAdapter 설정 시 자동 재시도

### 데이터 품질 관리

- **중복 감지**: 동일 timestamp/data_hash 체크
- **스키마 검증**: 예상 필드 누락 시 경고 로깅
- **타임스탬프 일관성**: 서버 응답 `created_at` vs 수집 시각 비교
