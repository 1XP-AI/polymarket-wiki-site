# 리베이트(마켓메이커)

> 한 줄 요약: Maker 리베이트 프로그램과 Liquidity Rewards의 구조, API 조회 방법, Copytrade 하이브리드 전략 적용법.

## 마켓 메이킹이란?

오더북에 **매수(Bid)와 매도(Ask) 주문을 동시에 걸어** 유동성을 제공하는 행위. Polymarket CLOB에서 Maker는 오더북에 resting order를 남기는 주체이며, Taker는 이를 가져가는 주체다.

Polymarket은 Maker에게 인센티브를 제공하여 오더북 깊이와 스프레드 경쟁력을 유도한다. Copytrade는 기본적으로 Taker 주문(수수료 지불)이므로, 리베이트 구조를 이해하면 **실행 비용 절감**에 직접 기여한다. ([[전략/Copytrade/실행-비용]] 참조)

## Maker Rebates Program

### 구조

- Maker 주문(오더북에 유동성 제공)에 대해 **일부 수수료를 돌려받음**
- Taker와 달리, 거래할 때마다 돈을 **버는** 구조
- 일일 USDC 기준으로 정산

### Rebate API 조회

```
GET /rebates/current?address={maker_address}&date={YYYY-MM-DD}
```

응답 예시:
```json
{
  "address": "0x...",
  "date": "2026-04-06",
  "rebate_usdc": 12.45,
  "volume": 5000.00
}
```

## Liquidity Rewards Program

특정 마켓에 유동성을 제공하면 **추가 보상**을 받을 수 있다. 리베이트와 별도로 운영되며 조건이 stricter.

### 보상 조건

- **최소 주문 크기**: 마켓별로 상이
- **스프레드 범위**: mid-price 기준 ±X% 이내
- **유지 시간**: order가 일정 시간 이상 오더북에 잔류

### Liquidity Rewards API

```
GET /rewards/markets/current          # 현재 보상 설정 조회
GET /rewards/markets/{condition_id}   # 특정 마켓 보상 상세
GET /rewards/user                     # 사용자 일별 보상 내역
GET /rewards/user/percentages         # 실시간 보상 비율
```

Copytrade에서 Liquidity Rewards를 활용하려면:
1. 카피 대상 트레이더가 주로 거래하는 마켓 확인
2. 해당 마켓의 현재 rewards 설정 조회 (`/rewards/markets/{condition_id}`)
3. 리베이트 예상액을 실행 비용에서 차감 → **순(Net) 카피 비용** 산출

## Copytrade 하이브리드 전략

Copytrade의 근본적 한계: **대부분 Taker 주문**이므로 수수료 비용이 누적된다.

### Maker-Taker 하이브리드

1. **카피 신호를 Taker로 받되, 반대側に Maker 주문 배치**
   - 트레이더가 YES를 사면 → NO 측에 Maker 주문 걸어두기
   - 스프레드를 벌면서 리베이트 수령

2. **제한가(Limit) 주문으로 진입**
   - 시장가(Market) 대신 제한가로 진입 시 Maker로 분류될 가능성 ↑
   - 단, 체결 지연 발생 → [[전략/Copytrade/매수-타이밍-전략]]에서 타이밍 손실 고려

3. **리베이트 임팩트 계산**
   - 예상 일일 카피 거래량 × Maker rebate rate = 일일 리베이트
   - 수수료 비용 - 리베이트 = 순 실행 비용

## 인벤토리 관리 (Inventory Risk)

마켓 메이커는 포지션이 한쪽으로 쏠리는 **인벤토리 리스크**를 관리해야 한다.

### Polymarket 특수성

- 예측 마켓은 결과가 **0 또는 1로 수렴** → 양방향 포지션 모두 만기 시 하나만 가치 있음
- 전통 주식 시장과 달리, 헷지가 완벽히 작동하지 않음
- 인벤토리 편향이 발생하면 빠르게 해소하거나 반대측 유동성 축소 필요

### 관리 규칙

1. **인벤토리 한도**: 단일 마켓에서 한쪽 방향으로 최대 $X 노출
2. **자동 헷지**: 인벤토리가 임계치 초과 시 반대측 주문 스프레드 확대
3. **재고 리밸런스**: 주기적으로 양 side 비율을 mid로 회귀

## 리베이트 vs 실행 비용 비교

| 항목 | Taker (일반 Copytrade) | Maker (리베이트 활용) |
|------|----------------------|---------------------|
| 수수료 | 지불 (0.1~0.5%) | 수령 (리베이트) |
| 체결 확률 | 즉시 | 조건부 (호가 도달 시) |
| 슬리피지 | 높음 | 낮음 (본인이 가격 설정) |
| 체결 지연 | 없음 | 발생 가능 |
| 인벤토리 리스크 | 낮음 | 높음 |
