# Copytrade: 리더보드 활용

> 한 줄 요약: 리더보드 데이터를 정량적 지표로 가공해 유망한 카피 대상(트레이더)을 선별하고, 위험을 통제하는 절차를 제시한다.

## 요약

리더보드는 Polymarket에서 활동이 증명된 트레이더 풀을 제공한다. 이 문서는 리더보드의 공개 필드(순위, 수익, 거래량, 거래 빈도 등)를 기반으로 copytrade 후보를 평가하는 방법론을 제안한다. 목표는 이후 심층 분석(프로필, 포지션, 과거 거래 패턴)으로 연결 가능한 후보를 효율적으로 걸러내는 것이다.

## 핵심 선별 지표

- 순위(rank): 장기적으로 높은 순위에 머문 트레이더 우선
- 누적 수익(profit): 높은 절대 수익과 안정적 수익 흐름을 확인
- 거래량(volume): 지나치게 낮은 거래량은 신호로서 약함
- 거래 빈도와 일관성: 일관된 활동은 전략 재현 가능성 증가
- 포지션 지속성: 빠른 진입·청산 반복은 추종 시 슬ippage 위험
- 마켓 분산성: 여러 마켓에 분산 투자하는 트레이더는 리스크 관리 능력 시사

## 선별 절차 (실무 가이드)

1. 리더보드(weekly/monthly)를 스냅샷으로 수집한다. ([[리더보드-트레이더-데이터]])
2. 상위 N명(예: top 50) 후보 추출 — 초깃값은 top 50 권장
3. 지표 표준화: 순위, 수익, 거래량을 z-score 또는 퍼센타일로 변환
4. 가중치 합산으로 스코어 산출(예: 0.4*profit + 0.3*volume + 0.3*consistency)
5. 상위 K명(예: 10명)을 대상으로 프로필/포지션 API 조회 및 정성검토
6. 백테스트 가능한 거래 패턴(진입·청산 타이밍, 평균 포지션 크기) 존재 여부 확인
7. 팔로우 우선순위 제정 및 리스크 한도(포지션당 자본 비중, 최대 누적 손실) 설정

## 예시 필터링 규칙

- 최소 거래량: 1,000 USDC 이상(지난 30일)
- 최소 거래 빈도: 주 1회 이상
- 최대 포지션 집중도: 특정 마켓에 50% 초과 보유 시 리스크 경고

## 리스크와 한계

- 데이터 지연: 리더보드·포지션 API의 지연/미반영 문제
- 행위자 식별 어려움: 같은 지갑이 여러 계정을 사용할 가능성
- 과거 수익은 미래 성과를 보장하지 않음

## 운영 사례 (예시)

사례: 2026-03월 상위 50 스냅샷에서 상위 12명을 선별한 뒤, 각 트레이더의 평균 포지션 지속시간과 마켓 분산성을 분석했다. 그 결과 A트레이더는 높은 승률을 보였으나 단일 마켓 집중도가 70%로 높아 리스크 허용치 초과로 제외되었고, B트레이더는 중간 수익·중간 거래량이었으나 다양한 마켓에 분산되어 샘플 추종 대상에 포함되었다. 실제 소규모 롤아웃(각 트레이더당 전체 자본의 1%씩)에서 누적 손실 한도에 도달한 사례는 없었으나, 리더보드 업데이트 지연으로 인한 포지션 불일치가 관찰되어 모니터링 규칙을 강화했다.

## 사이징 계산 예시

- 가정: 포트폴리오 자본 100,000 USDC, 최대포지션당 노출 2% → 포지션당 자본 = 2,000 USDC
- 트레이더 추종 비중 할당: 상위 트레이더에 대해 1~3% 범위(리스크·신뢰도에 따라 조정)
- 슬리피지·수수료 예측을 위해 예상 실행 비용 0.5%를 가정할 때, 단일 포지션의 기대 최대 손실 = 포지션당 자본 * (슬리피지 + 허용 손실 비율)
  예) 2,000 * 0.01(1% 허용 손실) = 20 USDC 기본 손실 허용
- 리스크 집합 규칙: 총 추종 노출(모든 트레이더 합산) ≤ 포트폴리오의 15%

## 백테스트 파라미터 샘플

- 기간: 최소 6~12개월(가능하면 여러 시장사이클 포함)
- 샘플 주기: 일일 수익 시계열 및 트레이더 진입/청산 이벤트 재구성
- 거래비용: 슬리피지 0.25~1.0%, 수수료 모델(고정 또는 비율)
- 리밸런싱 / 팔로우 지연: 실제 API 지연을 반영한 시뮬레이션(예: 1~10분 지연)
- 성능 지표: CAGR, MDD, Sharpe, Sortino, Win rate, 평균 트레이드 길이

## 구현 체크리스트

- [ ] 리더보드 스냅샷 스케줄러(주/월별)
- [ ] 데이터 정규화 파이프라인(지표 표준화, 결측치 처리)
- [ ] 가중치 기반 스코어링 모듈(파라미터 튜닝 가능)
- [ ] 백테스트 환경(거래비용, 지연 모델 포함)
- [ ] 운영 모니터링(포지션 불일치, 리더보드 변동 알림)
- [ ] 소규모 롤아웃 및 페이즈드 확장 정책
