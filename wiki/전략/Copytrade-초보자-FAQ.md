# Copytrade 초보자 FAQ

> 한 줄 요약: Copytrade(타인 거래 따라하기)를 시작할 때 자주 묻는 실무 질문과 권장 답변을 정리한 실전 안내서입니다.

## Q1. Copytrade를 바로 시작해도 되나요?

A: 직접 시작하기 전에 리스크 관리 규칙과 선별 체크리스트를 먼저 마련하세요. 특히 포지션 크기 제한, 스톱로스, 다수 트레이더 분산 복제 전략은 초기에 필수입니다. 단일 트레이더에 과도하게 의존하지 마세요.

## Q2. 어떤 트레이더를 따라야 하나요?

A: 정량적(수익률, 승률, 거래 빈도, MDD) + 정성적(마켓 선호, 활동 기간, 포지션 크기) 기준을 조합해 선별합니다. 리더보드 상위권은 후보가 되지만, 최근 활동과 일관성도 반드시 확인하세요. 자세한 선별 체크리스트는 [[Copytrade-초기-선별-가이드]]를 참고하세요.

## Q3. 실시간 복제는 어떻게 처리하나요(지연 문제)?

A: 감지 → 주문 생성 사이의 지연(latency)을 최소화하려면 가능한 한 API 폴링 대신 WebSocket 이벤트와 즉시 제출 가능한 주문 라우팅(사전검증된 가격 한도 주문)을 사용하세요. 플랫폼 구조와 실시간 데이터 처리 방법은 [[Polymarket-개요]]와 [[실시간데이터(WebSocket)]] 문서를 함께 확인해 구현 세부사항을 검토하세요. 슬리피지와 유동성 문제를 고려해 진입 크기를 작게 설정하는 것이 유리합니다.

## Q4. 수수료와 슬리피지 때문에 수익이 감소하는데요?

A: 예: 카피 시 대부분 테이커 수수료를 내게 되므로, 수수료와 예상 슬리피지를 반영한 비용 모델을 만들어 기대수익을 조정해야 합니다. 작은 포지션으로 여러 트레이더를 분산하면 비용 대비 이득을 안정화할 수 있습니다. 관련 내용은 [[실행-비용]] 및 [[슬리피지]] 문서를 참조하세요.

## Q5. 트레이더가 갑자기 전략을 바꾸면?

A: 자동 모니터링(성과 점수 하락, 포지션 패턴 변화 감지)을 통해 일정 임계치(예: 누적 수익률 -10% 또는 승률 20% 감소) 이하로 내려가면 자동으로 팔로우를 중단하도록 설정하세요. 또한 오프라인(수작업)으로 정기 검토를 권장합니다.

## Q6. 초보자가 실전에서 실수하기 쉬운 부분은?

A: 1) 포지션 과다집중 2) 수수료·슬리피지 미반영 3) 표본이 적은 트레이더 추종 4) 백테스트 없이 실거래 적용 — 이 네 가지를 경계하세요.

## Q7. 샘플 워크플로(초보자용)

1) 리더보드 스냅샷 수집(주간)
2) 상위 50명 필터링 → 지표 표준화(z-score)
3) 상위 10명으로 압축 후 프로필·포지션 API 조회
4) 샌드박스에서 100회 백테스트(슬리피지 0.5%, 지연 1~5분 가정)
5) 소규모 롤아웃(각 트레이더에 포트폴리오의 0.5~1.0% 할당)
6) 실시간 모니터링(포지션 불일치, 누적손실 임계치) → 자동 중단

## 실행 템플릿(의사코드)

- 이벤트 수신: WebSocket 리더보드/트레이더 포지션 업데이트
- 필터링: 최근 30일 거래량 > threshold AND activity_rate > 0
- 스코어링: score = 0.4*profit_z + 0.3*volume_z + 0.3*consistency_z
- 백테스트: 비용모델(slippage_pct, fee_pct), delay_seconds
- 운영: if score > X and backtest_pass then enable_follow

(구현 참고: [[API-예제-샘플]], [[API-레퍼런스]], [[리더보드-트레이더-데이터]])

## 샌드박스 백테스트 결과 해석(예)

- 기대값: CAGR 12%, MDD 8% 이면 합리적
- 실패신호: 백테스트에서 리밸런싱/지연을 반영했을 때 성능이 급락하면(예: CAGR -5% → 제외)
- 노이즈 확인: 소규모 트레이더(거래건수 < 50)에서 나온 우연적 성과는 신뢰 낮음

## 빠른 체크리스트(요약)

- 리더보드 상위 50 스냅샷 확보
- 상위 10명에 대해 포지션·프로필 추가조회
- 백테스트(거래비용·지연 포함) 수행
- 포지션당 노출 제한 설정(예: 1~2% 목표)
- 자동 중단 규칙 구성(성과·포지션 불일치)
