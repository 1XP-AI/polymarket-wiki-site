# 트레이더 프로필 분석

> 리더보드에서 발견된 트레이더를 copytrade 대상으로 평가하는 기준과 분석 템플릿

## 개요

이 문서는 Polymarket 리더보드에서 상위 트레이더를 발굴하여 "팔로우 가치"를 정량·정성으로 평가하는 템플릿을 제공합니다. 목표는 copytrade에서의 리스크를 감소시키고, 자동화된 선별 파이프라인에 투입 가능한 명확한 지표를 만드는 것입니다.

## 분석 항목 (확장)

1. 기본정보
   - 활동 기간, 지갑주소, 공개 소셜(트위터 등), 계정 생성일.
2. 성과지표
   - 누적 수익률, 승률(종료된 포지션 기준), 평균 포지션 보유기간, 샤프 지수(가능한 경우).
3. 포지션 패턴
   - 선호 마켓(예: 크립토, 스포츠), 평균 포지션 크기, 동일 마켓 반복 거래 비율, 진입/청산 시간대 분포.
4. 리스크 프로필
   - 포지션 규모 대비 계정 잔액(추정), 최대 드로다운, 레인지 비중, 헷지 사용 여부.
5. 재현가능성
   - 전략이 규칙(룰)으로 표현 가능한지 여부(예: 이벤트 기반, 가격 기반, 시간 기반).
6. 소셜·신뢰성
   - 공개 발언의 전문성, 전략의 일관성, 외부 리포트·블로그 연계성.
7. 행동 안정성 (추가)
   - 최근 N일(예: 30일) 내 활동 일관성, 대규모 포지션의 빈도.

## 평가 템플릿 (구체화)

- 팔로우 점수(0-100) = 0.4 * 성과점수 + 0.3 * 리스크점수 + 0.2 * 재현가능성 + 0.1 * 신뢰성
  - 성과점수: 승률(종료된 포지션 기준), 누적수익률, 샤프비율을 가중합으로 변환
  - 리스크점수: 최대 드로다운·평균 손실 대비 최대손실 비율 역수 등을 사용
  - 재현가능성: 규칙화 가능한 전략이면 높은 점수
  - 신뢰성: 공개 소셜·외부 근거의 존재 여부

- 최소 요건(필터)
  - 승률 >= 50% (최근 90일 기준)
  - 평균 포지션 보유기간이 지나치게 짧거나 긴 경우엔 재검토
  - 계정에 비해 포지션 규모가 지나치게 큰 트레이더는 리스크 조정 필요

## 데이터 기반 지표 계산 예제

1) 승률 계산 (종료된 포지션 기준)

- API: GET https://data-api.polymarket.com/closed-positions?address={wallet}
- 로직: 종료된 포지션에서 PnL > 0 비율

예: curl을 이용한 간단한 수집

```bash
curl -s "https://data-api.polymarket.com/closed-positions?address=0xABC..." | jq '.[] | select(.pnl > 0) | .id' | wc -l
```

2) 평균 포지션 크기 및 보유기간

- API: GET https://data-api.polymarket.com/positions?address={wallet}
- 보유기간: 포지션 생성 시간 ~ 청산 시간(청산된 경우)

3) 팔로우 점수 계산 (파이썬 의사코드)

```python
# pseudo
metrics = {
  'win_rate': 0.6,
  'sharpe': 1.2,
  'max_drawdown': 0.12,
  'rule_score': 0.8,
  'social_score': 0.7,
}
score = 0.4 * (metrics['win_rate']*100 * 0.6 + metrics['sharpe']*10 * 0.4) \
      + 0.3 * (1 - metrics['max_drawdown']) \
      + 0.2 * metrics['rule_score']*100 \
      + 0.1 * metrics['social_score']*100

# normalize to 0-100
score = min(max(score, 0), 100)
```

## 자동화 템플릿

- 주기: 일간/주간으로 리더보드 수집 → 새로운 트레이더 발견 시 자동으로 프로필 폴더 생성
- 데이터 수집: closed-positions, positions, trades, activity API에서 JSON 수집 후 raw/에 저장
- 문서 생성: 자동 요약(한줄), 주요 지표 표, 예비 팔로우 점수 계산 결과를 wiki/트레이더/{이름}/프로필.md에 추가

## 적용 예시

- 리더보드 상위 30명에 대해 90일 데이터를 백테스트하여 팔로우 점수 상위 10명을 후보군으로 선정
- 후보군을 대상으로 소규모 실거래(미러 포지션 크기 1-5%)를 통해 실성능 검증 후 스케일업

## 품질 점검 항목

- 데이터 소스가 정상 응답하는지(HTTP 200)
- closed-positions와 trades 간의 불일치 확인(데이터 품질 문제)
- 최근 활동이 없는 트레이더는 자동 필터링

## 실행 체크리스트

- [ ] 후보 트레이더 기본 식별 완료
- [ ] 포지션 히스토리 수집 및 패턴 확인
- [ ] 성과 지표(승률/수익) 검증 완료
- [ ] 전략 일관성 및 리스크 관리 여부 확인
- [ ] 소셜 발언과 포지션 일치 여부 크로스체크
- [ ] 온체인/지갑 매핑(가능한 경우) 완료
- [ ] 예상 드로다운 및 실행비용(슬리피지 포함) 문서화

## 추가: 검증 단계

실무에서 바로 적용 가능한 단계별 검증:

1. **기본 식별**
   - 트레이더명(또는 지갑주소) 기록
   - 공개된 소셜/프로필 링크 수집 (트위터/X, GitHub, 블로그)
   - 활동 기간 및 빈도 확인 (최근 30/90일 활동 여부)

2. **포지션 신뢰성**
   - 리더보드/프로필에서 확인되는 포지션 이력 추출
   - 동일 포지션의 반복성(패턴) 확인: 1회성 베팅인지 지속적 전략인지
   - 포지션 진입·청산 타이밍의 일관성(예: 이벤트 발표 전후 행동)

3. **성과 지표**
   - 승률, 누적 수익률(가능한 범위 내에서), 손익의 분포 확인
   - 큰 수익/손실 사례의 원인 분석(리스크 테이크인지 운이었는지)
   - 샘플 기간 및 샘플 사이즈 신뢰성 검토

4. **전략적 일관성**
   - 트레이더가 선호하는 마켓(정치/경제/스포츠/크립토) 확인
   - 포지션 크기/사이즈 규칙(비례적 포지션 vs 전형적 베팅) 파악
   - 리스크 관리(스톱·리밸런싱·헤징) 관련 언급 여부

5. **공개 발언과 행동의 정합성**
   - 트위터/포럼에서 한 발언이 실제 포지션과 일치하는지 크로스체크
   - 자주 전략을 바꾸는지, 혹은 특정 이벤트에서 일관된 성향을 보이는지 분석

6. **온체인/프로필 검증**
   - 가능하면 지갑주소와 공개 프로필 연결(온체인 활동 로그 확인)
   - 동일 지갑이 다수의 계정에 의해 공유되지 않는지 확인

7. **리스크 노트**
   - 팔로우 시 예상되는 최대 드로다운(과거 패턴 기준) 추정
   - 유동성·슬리피지 위험 — 대형 포지션이 시장 가격에 미치는 영향 추정
   - 규제·계정 리스크(예: 외부 커뮤니케이션에 취약한 경우)
