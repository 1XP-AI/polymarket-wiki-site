# 리스크 알림 규칙

> 한 줄 요약: Copytrade 운영 중 위험 상태 발생 시 자동으로 알림을 트리거하고 대응하는 규칙 모음

## 내용

Copytrade는 여러 트레이더를 동시에 추종할 때 누적 리스크가 빠르게 증가할 수 있다. 이 문서는 운영자가 실시간으로 인지하고 조치할 수 있도록 설계된 알림 규칙을 정리한다.

주요 알림 트리거:

1. 포지션 손실 임계치 초과: 포트폴리오 전체 손실이 사전 정의된 임계치(예: -3% 단일 사이클, -10% 누적)를 넘으면 즉시 알림.
2. 단일 트레이더 이상 집중: 특정 트레이더의 노출이 전체 포트폴리오의 N%를 넘으면 경고.
3. 급격한 포지션 변화: 리더보드 상위 트레이더가 1시간 내 포지션을 대폭 변경(예: 포지션 규모 2배 이상)하면 알림.
4. 유동성 경고: 목표 마켓의 주문집중도 또는 예상 슬리피지 증가 시 알림.
5. 데이터 소스 실패: API 응답 실패 또는 데이터 지연이 감지되면 운영자에게 통보.

알림 우선순위와 조치:

- P0 (즉시): 포트폴리오 손실 임계치 초과 — 자동 추종 중지(임시) + 운영자 이메일/슬랙 알림
- P1 (높음): 단일 트레이더 집중 초과 — 해당 트레이더 비중 축소 권고
- P2 (중): 데이터 소스 불안정 — 수집 재시도 및 모니터링

## 구현 고려사항

- 알림은 취소 가능한 자동화(예: 임시 추종 중지)와 사람 개입을 연계한다.
- 각 경보는 raw 데이터 또는 Polymarket API에서 검증 가능한 근거를 포함해야 한다.
