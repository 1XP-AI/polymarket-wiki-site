# Copytrade 심화

> Polymarket에서 따라하기(copytrade)를 실전으로 옮기기 위한 심화 가이드

## 핵심 요약

이 문서는 Polymarket에서의 copytrade 전략을 심화해 다룹니다. 주요 내용은(1) 트레이더 선택 기준, (2) 리스크 관리 프레임, (3) 포지션 진입·청산 타이밍, (4) 실무용 API 예제입니다. 원본 시드는 raw/03-Copytrade-전략분석.md입니다.

## 1. 트레이더 선택 기준

- 리더보드 상위권 활동 여부와 평균 수익률
- 포지션 규모(Gross exposure)와 포지션 빈도
- 전략의 일관성(예: 이벤트 기반, 장기 베팅 등)

우선순위: 리더보드 증거 기반 > 포지션 패턴 일관성 > 소셜·주석(보조)

## 2. 리스크 관리(요약)

자세한 리스크 관리 기법은 [[리스크-관리]] 문서를 참고하되, copytrade 관점에서의 핵심은:

- 포지션 크기 상한: 계좌 대비 % 노출 제한
- 손절/청산 규칙: 확률 변화(odds) 기준으로 자동 청산 트리거 설정
- 다중 트레이더 분산: 단일 트레이더 편중 방지

예시: 계좌의 2% 초과 포지션 금지, 트레이더별 최대 20% 노출

## 3. 진입/청산 타이밍 패턴

- 트리거형 진입: 트레이더가 특정 권역(예: 0.6 이상)에서 반복 진입하는 경우
- 모멘텀 기반 청산: 오즈가 빠르게 역이동할 때 부분 청산
- 이벤트 전후 조정: 이벤트(예: 경기 시작, 투표 마감) 직전 보수적 포지션 축소

## 4. 실무용 API 예제

다음은 시장 목록을 가져와 특정 트레이더 포지션을 조회하는 간단한 예시(curl):

```bash
# 활성 시장 리스트(예시)
curl -s "https://gamma-api.polymarket.com/markets?limit=20" | jq '.'

# 리더보드에서 발견한 지갑 주소의 프로필/포지션 조회(예시)
curl -s "https://gamma-api.polymarket.com/profiles/{address}"
```

실제 구현 시에는 rate limit을 지키고, 오류 시 재시도/백오프 로직을 넣어야 합니다.
