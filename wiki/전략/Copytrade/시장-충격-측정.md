# 시장 충격 측정

> Copytrade 실행 시 시장 충격(market impact)을 정의하고, 간단한 측정 방법과 완화 아이디어를 제시합니다.

## 시장 충격의 정의

시장 충격(market impact)은 대규모 매수/매도 또는 연속 주문이 호가에 미치는 영향과 이후 가격 복구 패턴을 의미합니다. Copytrade에서는 리드 트레이더의 포지션 크기가 클수록, 그리고 추종자들이 동시에 진입할수록 시장 충격이 증폭됩니다. 특히 마켓당 유동성이 $50K 미만인 경우, $500 이상의 단일 주문만으로도 스프레드가 크게 벌어질 수 있습니다.

## 시장 충격의 두 가지 페이즈

### 1. 즉시 충격 (Immediate Impact)

주문이 체결되는 순간 발생하는 가격 이동입니다. 오더북의 얇은 레벨을 먹어치우며 Best Ask/Bid가 멀어집니다.

| 주문 크기 | 유동성 $10K 마켓 | 유동성 $50K 마켓 | 유동성 $200K+ 마켓 |
|----------|-----------------|-----------------|-------------------|
| $50 | 0.5-1.0¢ 이동 | 0.1-0.3¢ 이동 | 무시 가능 |
| $200 | 2-5¢ 이동 | 0.5-1.0¢ 이동 | 0.1-0.3¢ 이동 |
| $1,000 | 10-20¢ 이동 | 3-5¢ 이동 | 0.5-1.5¢ 이동 |

즉시 충격은 [[전략/Copytrade/실행-비용]]의 주요 구성 요소로, 슬리피지 비용에 직접 반영됩니다.

### 2. 잔류 충격 (Permanent Impact)

주문 이후에도 가격이 원래 수준으로 돌아오지 않는 현상입니다. 이는 시장이 주문의 정보를 반영했기 때문일 수 있으며, Copytrade에서는 추종자들이 리드 트레이더의 진입 신호를 보고 함께 진입하면서 잔류 충격이 더 커집니다.

## 측정 지표

### Impact Cost

```
Impact Cost = (체결가 - 주문 전 Mid Price) / 주문 전 Mid Price
```

- Buy 주문: 체결가가 Mid보다 높을수록 양수 (불리)
- Sell 주문: 체결가가 Mid보다 낮을수록 음수 (불리)

### 가격 복구 지수 (Price Reversion Index)

```
PRI = (충격 후 가격 - N분 후 가격) / (충격 후 가격 - 주문 전 Mid Price)
```

- PRI ≈ 1.0: 완전 복구 (충격이 일시적, 유동성 충분)
- PRI ≈ 0.0: 복구 없음 (충격이 시장 정보로 흡수됨)
- PRI < 0: 추가 이동 (충격이 연쇄 반응 유발)

### 주문 크기 대비 즉시 슬리피지

체결 시점에서 기대 가격과 실제 체결가의 차이를 주문 크기로 나누어 정규화합니다.

## 측정 절차

1. **데이터 수집**: Polymarket API([[개념/CLOB]])에서 대상 마켓의 과거 체결 로그(trade history)와 오더북 스냅샷 수집
2. **기준가 설정**: 각 체결 전 마지막 mid price = (best bid + best ask) / 2
3. **충격 계산**: 체결가와 기준가 비교 → Impact Cost 산출
4. **분위별 분석**: 주문 크기를 소/중/대로 구분하여 평균 슬리피지 계산
5. **추이 관찰**: 체결 후 1분, 5분, 15분 지점에서 가격 비교 → 가격 복구 지수 계산

### 샘플 측정 쿼리

```bash
# 특정 마켓의 최근 체결 내역
curl -s "https://gamma-api.polymarket.com/trades?market=${MARKET_ID}&limit=100" \
  | jq '.[] | {price: .price, size: .size, timestamp: .timestamp}'

# 오더북 스냅샷 (체결 전후 비교용)
curl -s "https://gamma-api.polymarket.com/book?market=${MARKET_ID}" \
  | jq '{best_bid: .bids[0].price, best_ask: .asks[0].price}'
```

## 완화 전략

### 주문 분할 (Time-Weighted Execution)

대형 주문을 여러 소규모 주문으로 나누어 시간을 두고 실행합니다.

- **TWAP**(Time-Weighted Average Price): 고정 간격으로 분할 주문
- **VWAP**(Volume-Weighted Average Price): 시장 거래량 패턴에 맞춰 분할
- Copytrade 컨텍스트: 리드 트레이더의 한 포지션에 대해 3~5로 분할해 5-15분에 걸쳐 실행

### 가격 민감도 기반 포지션 조절

마켓별로 Impact Cost를 미리 계산해, 임계치를 초과하는 경우:

1. 포지션 크기를 줄이고 잔액은 [[개념/슬리피지]]가 낮은 마켓으로 분산
2. 해당 트레이더의 신호를 "실행 불가"로 분류하고 다음 주기에 재평가
3. [[전략/Copytrade/추종-중단-규칙]]과 연계: 충격이 지속되는 트레이더는 일시 제외

### 대체 신호 평균화

여러 트레이더가 동일한 마켓에 진입할 때, 평균 진입 가격을 계산하고 가장 효율적인 타이밍에 실행합니다. 중복 신호를 한 번의 주문으로 통합하면 시장 충격을 최소화할 수 있습니다.

## 시장 충격과 Copytrade의 악순환

Copytrade의 구조적 문제가 시장 충격을 악화시킬 수 있습니다:

```
리드 트레이더 진입 → 가격 상승 → 추종자 1 진입 → 추가 상승
  → 추종자 2+N 진입 → 가격 급등 → 평균 진입가 악화
  → 전량 진입 완료 시 이미 불리한 가격 → 기대 수익 하락
```

이 악순환을 끊기 위해:
- [[전략/Copytrade/추종-실행-속도-최적화]]에서 진입 지연을 전략적으로 활용
- 시장 충격이 큰 마켓에서는 Copytrade 대신 직접 분석 기반 진입 고려
- [[전략/Copytrade/알고리즘-선별-체크리스트]]에 유동성/충격 임계값 포함
