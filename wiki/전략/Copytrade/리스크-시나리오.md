# 리스크-시나리오

> Copytrade 수행 시 고려해야 할 주요 리스크 시나리오와 대응 방안 요약

## 내용

이 문서는 Copytrade(타 트레이더를 따라 거래) 전략을 실행할 때 실제로 마주칠 수 있는 주요 리스크 시나리오들을 정리하고, 각 시나리오에 대한 실용적 대응 방안을 제시합니다. 목표는 자동화된 추종(복사 거래) 환경에서 손실을 최소화하고 운영 안정성을 높이는 것입니다.

### 1) 트레이더 오류(실수성 거래)
- 시나리오: 리더(trader)가 실수로 대량 매수를 하는 등 비정상적 포지션을 취함.
- 영향: 복사 거래자가 동일한 거래를 따라가면 큰 손실 발생.
- 대응:
  - 거래 임계값 설정: 단건 거래 크기 또는 포지션 변화율이 일정 임계값(예: 계정 자산의 5%)을 넘으면 자동 차단.
  - 휴먼 확인: 임계값 초과 시 알림을 보내고 수동 확인 후 진행.

### 2) 시장 유동성 붕괴 / 급격한 슬리피지
- 시나리오: 얕은 마켓에서 대규모 주문으로 슬리피지 발생, 체결 가격이 크게 불리해짐.
- 영향: 실행 비용 급증, 예상 수익 소멸.
- 대응:
  - 슬리피지 가산: 예상 슬리피지 모델을 적용해 추종 비율을 조정.
  - 포지션 크기 제한: 얕은 마켓에서는 추종 비율을 낮춤.
  - 분할 실행: 주문을 여러 분할로 실행해 시장 충격 완화.

### 3) 트레이더 탈퇴 또는 중단
- 시나리오: 추종 대상 트레이더가 활동을 멈추거나 계정 접근을 상실.
- 영향: 자동 전략이 유효하지 않게 되며 추종자는 방향을 잃음.
- 대응:
  - 헬스 체크: 트레이더의 최근 활동을 주기적으로 점검하여 활동 중지가 감지되면 자동으로 추종 중단.
  - 백업 트레이더: 주요 트레이더가 중단되면 미리 선정한 대체 트레이더 풀로 전환.

### 4) API 장애 또는 데이터 지연
- 시나리오: Polymarket API 또는 데이터 파이프라인의 지연/오류로 잘못된 신호가 들어옴.
- 영향: 시차에 의한 잘못된 실행, 중복 주문 등.
- 대응:
  - 타임아웃/재시도 정책: API 응답 지연 시 보수적으로 재시도하거나 작업 중단.
  - 다중 소스 검증: 중요한 신호는 2개 이상 소스에서 교차검증.

### 5) 전략적 리스크(집중 위험)
- 시나리오: 여러 추종자가 동일 트레이더를 동시에 따라 대량 포지션 집중.
- 영향: 플랫폼 수준의 시장 충격 유발 가능성.
- 대응:
  - 포지션 상한: 추종자가 시장 내 전체 노출의 일정 비율을 넘지 않도록 제한.
  - 분산 추종: 동일 트레이더 외에 보완 트레이더를 혼합하여 리스크 분산.

## 적용 가이드
- 각 시나리오별로 자동화 규칙(임계값, 타임아웃, 알림)을 설정하고, 운영 로그를 통해 예외 사례를 지속 학습한다.
- 백테스트와 시뮬레이션을 통해 각 정책의 기대 손실 감소 효과를 검증한다.
