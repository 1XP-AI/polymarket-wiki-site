# 팔로우-스코어링 모델

> 트레이더를 팔로우할지 결정하기 위한 계량화된 점수 모델 구조와 구현 가이드

## 개요

팔로우-스코어링은 리더보드나 온체인/오프체인 데이터를 바탕으로 특정 트레이더를 "팔로우 가치"로 정량화하는 절차입니다. 본 문서는 최소한의 입력(승률, 평균 베팅 크기, 유동성 민감도, 리스크 노출 등)을 받아 팔로우 여부와 노출 크기를 추천하는 모델을 제시합니다.

## 핵심 변수

| 변수 | 기호 | 의미 | 데이터 소스 |
|------|------|------|-------------|
| 승률(WinRate) | WR | 지난 N건 중 정답 비율 | 리더보드, 포지션 히스토리 |
| 평균 베팅 크기 | AvgSize | 트레이더 평균 베팅 규모(USD) | 트레이더 포지션 API |
| 변동성 민감도 | VolSens | 시장 변동성에 따른 포지션 변동성 | 30일 포지션 StdDev |
| 청산/오버나이트 리스크 | LiquidRisk | 청산 빈도 또는 급격한 포지션 변경 비율 | 포지션 스냅샷 |
| 행동 일관성 | Consistency | 진입/청산 패턴의 일관성 지표 | 거래 히스토리 분석 |
| 샘플 크기 | N | 관측 거래 수 | 리더보드 기간 필터 |
| 최근성 | Recency | 최근 활동 집중도 (EMA 가중) | 거래 타임스탬프 |

## 점수화 공식

### 기본 모델

```
FollowScore = w1·Norm(WinRate) + w2·Norm(Consistency) + w3·Norm(Recency)
            - w4·Norm(LiquidRisk) - w5·Norm(VolSens)
```

### 가중치 프로파일 (사용자 성향별)

| 변수 | 보수적(Conservative) | 균형(Balanced) | 공격적(Aggressive) |
|------|---------------------|----------------|-------------------|
| w1 (WinRate) | 0.35 | 0.25 | 0.20 |
| w2 (Consistency) | 0.25 | 0.20 | 0.15 |
| w3 (Recency) | 0.10 | 0.20 | 0.25 |
| w4 (LiquidRisk) | 0.20 | 0.25 | 0.25 |
| w5 (VolSens) | 0.10 | 0.10 | 0.15 |

- **보수적**: 리스크 항(w4, w5)이 크고, 과거 일관성에 높은 가중치
- **균형**: 5개 변수를 고르게 배분 — 기본 권장
- **공격적**: 최근성과 승률을 우선 — 단기 변동성 높은 트레이더에도 진출

### 정규화 (Min-Max + Laplace Smoothing)

```
Norm(x) = (x - min_x) / (max_x - min_x)     # 후보군 내에서 0~1 정규화
LaplaceWinRate = (wins + 1) / (total + 2)   # N < 15 시 편향 보정
```

N이 작은 표본(특히 신규 트레이더)은 Laplace smoothing 없이 승률이 100% 또는 0%로 왜곡되기 쉽다. 15회 미만 거래 트레이더는 반드시 보정 적용.

## 구현 (Python)

```python
import numpy as np
import pandas as pd
from scipy.stats import pearsonr
from datetime import datetime, timedelta

def laplace_smooth(wins, total, alpha=1.0):
    """N이 작을 때 승률 편향 보정"""
    return (wins + alpha) / (total + 2 * alpha)

def minmax_normalize(series):
    r = series - series.min()
    d = series.max() - series.min()
    return r / d if d > 0 else pd.Series(0.5, index=series.index)

def compute_follow_score(df, profile='balanced'):
    """
    트레이더 DataFrame에 FollowScore 컬럼 추가.

    Parameters
    ----------
    df : pd.DataFrame
        필요한 컬럼: win_rate, consistency, recency_ema,
                      liquid_risk, vol_sens, trades_30d
    profile : str
        'conservative', 'balanced', 'aggressive'

    Returns
    -------
    pd.DataFrame (score 컬럼 추가)
    """
    weights = {
        'conservative': {'wr': 0.35, 'cons': 0.25, 'rec': 0.10,
                         'liq': 0.20, 'vol': 0.10},
        'balanced':     {'wr': 0.25, 'cons': 0.20, 'rec': 0.20,
                         'liq': 0.25, 'vol': 0.10},
        'aggressive':   {'wr': 0.20, 'cons': 0.15, 'rec': 0.25,
                         'liq': 0.25, 'vol': 0.15},
    }
    w = weights.get(profile, weights['balanced'])

    # Laplace smoothing for small samples
    mask = df['trades_30d'] < 15
    df.loc[mask, 'win_rate'] = df.loc[mask].apply(
        lambda r: laplace_smooth(
            r['win_rate'] * r['trades_30d'],
            r['trades_30d']
        ), axis=1
    )

    # Normalize all inputs to [0,1]
    scores = pd.DataFrame()
    scores['wr']   = minmax_normalize(df['win_rate'])
    scores['cons'] = minmax_normalize(df['consistency'])
    scores['rec']  = minmax_normalize(df['recency_ema'])
    scores['liq']  = 1 - minmax_normalize(df['liquid_risk'])   # 음수 방향
    scores['vol']  = 1 - minmax_normalize(df['vol_sens'])       # 음수 방향

    # Weighted sum
    follow_score = (w['wr']  * scores['wr'] +
                    w['cons'] * scores['cons'] +
                    w['rec']  * scores['rec'] +
                    w['liq']  * scores['liq'] +
                    w['vol']  * scores['vol'])

    df['follow_score'] = (follow_score * 100).round(2)  # 0~100 스케일
    return df
```

## 상관관계 페널티

동일한 마켓/전략을 선호하는 트레이더들을 동시에 팔로우하면 분산 효과가 없다. FollowScore 계산 후 상관계수로 보정한다.

```python
def apply_correlation_penalty(df, returns_dict):
    """
    높은 상관관계를 가진 트레이더의 score를 감산.

    Parameters
    ----------
    df : pd.DataFrame (follow_score 컬럼 포함)
    returns_dict : dict[str, pd.Series]
        trader_id → 일별 수익률 시계열

    Returns
    -------
    pd.DataFrame (adjusted_score 컬럼 추가)
    """
    traders = list(df.index)
    adjusted = df['follow_score'].copy()

    for i in traders:
        if i not in returns_dict:
            continue
        corrs = []
        for j in traders:
            if j == i or j not in returns_dict:
                continue
            aligned = pd.concat([returns_dict[i], returns_dict[j]], axis=1).dropna()
            if len(aligned) < 20:
                continue
            r, _ = pearsonr(aligned.iloc[:, 0], aligned.iloc[:, 1])
            corrs.append(r)
        if corrs:
            max_corr = max(corrs)
            adjusted[i] *= max(0.3, 1 - max_corr)  # 최소 30% 유지

    df['adjusted_score'] = adjusted.round(2)
    return df
```

## 적용 절차

1. **데이터 수집**: 리더보드와 포지션 히스토리로 입력 변수 산출([[트레이더/리더보드/데이터]] 참조)
2. **Laplace 보정**: N < 15 트레이더 승률 smoothing
3. **정규화**: 후보군 내 Min-Max 0~1 스케일링
4. **가중치 적용**: 프로필(보수적/균형/공격적) 선택 → FollowScore 계산
5. **상관관계 페널티**: 높은 상관 trader score 감산
6. **최종 정규화**: `final_weight = adjusted_score / sum(all adjusted_scores)`
7. **임계값 컷오프**: FollowScore ≥ 60점 + AdjustedScore ≥ 40점 후보 선별

## 임계값과 의사결정 매트릭스

| FollowScore | AdjustedScore | 행동 | 권장 노출 |
|-------------|---------------|------|-----------|
| ≥ 80 | ≥ 60 | 즉시 팔로우 | 자본 2~5% |
| 60~79 | ≥ 40 | 조건부 팔로우 | 자본 1~2% |
| 40~59 | — | 관찰 목록 | 0% (모니터링만) |
| < 40 | — | 제외 | 0% |

## 한계와 주의사항

- **데이터 편향**: 리더보드 퍼포먼스는 생존자 편향(survivorship bias) 포함. 중도 하차한 트레이더 데이터는 누락됨.
- **과거 ≠ 미래**: 비정상적 이벤트(정보 유출, 대형 론칭)는 별도 필터 필요. [[전략/Copytrade/리스크-관리]] 참조.
- **과대적합**: 소수 샘플에서 높은 FollowScore 나온 트레이더 — 반드시 백테스트로 교차검증([[개념/Copytrade#백테스트 설계 (권장)]]).
- **상관관계 드리프트**: 두 트레이더의 과거 상관이 미래에도 유지된다는 보장 없음. 주기적 재계산 권장.
- **플랫폼 변화**: Polymarket API 변경, 데이터 소스 중단 시 파이프라인 전체 재검토 필요.

## 모델 개선 주기

| 단계 | 빈도 | 작업 |
|------|------|------|
| score 재계산 | 주 1회 | 최신 30일 데이터로 FollowScore 갱신 |
| 가중치 튜닝 | 월 1회 | 과거 90일 백테스트 결과로 optimal weight 탐색 |
| 상관관계 갱신 | 주 1회 | returns_dict 최신화 → penalty 재적용 |
| 모델漂移 감지 | 분기 1회 | IC(Information Coefficient) = score와 실제수익 간 상관계수 추적. IC < 0.1 이면 모델 재설계 고려 |
