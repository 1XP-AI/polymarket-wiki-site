# 성과 분석 지표 (Copytrade 성능 메트릭)

> Copytrade 대상의 성과를 계량적으로 평가하고 비교하기 위한 핵심 지표, 계산법, 실제 적용 패턴을 정리한다.

## 핵심 원칙

Copytrade에서는 단순한 승률만으로 트레이더를 평가하면 오판할 위험이 크다. 고승률 + 저거래수 트레이더는 우연에 의한 결과일 수 있으며, 리더보드의 기간별 순위는 특정 마켓 이벤트에 편향될 수 있다. 아래 지표들을 **복합적**으로 사용해 종합적으로 성과를 판단해야 한다.

## 핵심 지표 상세

### 1. 수익률 계열

| 지표 | 계산식 | 해석 |
|------|--------|------|
| 순수익(ROI) | 기간수익 / 평균포지션규모 | 절대적 수익 수준 |
| 연환산화 수익률 | `(1 + ROI)^(365/일수) - 1` | 기간 차이 보정 |
| 누적수익곡선 | 일자별 누적 PnL 시계열 | 지속성 시각화 |

### 2. 리스크-조정 지표

| 지표 | 계산식 | 권장 임계값 |
|------|--------|-------------|
| 변동성(Volatility) | `std(포지션별수익률)` | 낮을수록 안정적 |
| 샤프 유사 비율 | `(평균수익 - 시장평균) / 수익Std` | >1.0 권장 |
| Sortino 비율 | `(평균수익 - 시장평균) / downside_std` | >1.2 권장 |
| 최대 낙폭(MDD) | `(최저점 - 최고점) / 최고점` | >15% 면 경고 |
| Calmar 비율 | `연환산화수익 / abs(MDD)` | >2.0 우수 |

Prediction market 특성상 "무위험이자율" 대신 **시장 평균 수익률**을 베이스라인으로 사용한다. 예를 들어 상위 50개 마켓의 평균 수익률이 5%라면, 이를 무위험수익 대용으로 차감한다.

### 3. 행동 패턴 지표

| 지표 | 측정 | 활용 |
|------|------|------|
| 승률(Win Rate) | 수익거래 / 총거래 | 기본 필터, 단독사용 금지 |
| 기대값(EV) | `승률×평균이익 - (1-승률)×평균손실` | EV > 0 이면 장기 흑자 기대 |
| 홀딩 타임 | 평균 `청산시간 - 진입시간` | 단타vs장기 분류 |
| 거래 빈도 | 30일당 거래건수 | 실행비용 민감도 판단 |
| 평균 베팅 사이즈 | `총베팅액 / 거래건수` | 자본 요구량 추정 |
| 연속승/패 스크 streak | 최장연승, 최장연패 | 심리적 압박도 측정 |

### 4. 실행 품질 지표

| 지표 | 측정 | 중요성 |
|------|------|--------|
| 부분체결 비율 | 부분체결건 / 총주문건 | >30% 면 유동성 경고 |
| 평균 슬리피지 | `체결가 - 기대가` | [[개념/슬리피지]] 참고 |
| 시장 충격비용 | 대형주문 전후 가격 차이 | [[전략/Copytrade/시장-충격-측정]] |

## 지표 통합: 트레이더 점수화

각 지표를 z-score로 표준화 후 가중합하여 종합 점수를 산출한다.

```python
def compute_trader_score(metrics: dict, weights: dict) -> float:
    """
    트레이더 종합 점수 계산
    
    metrics keys: win_rate, sharpe, mdd, volume, recency, holding_days
    weights keys: 동일 (총합 1.0)
    """
    # 각 지표를 0~1로 정규화 (population 기준 min-max)
    normalized = {}
    normalized['win_rate'] = max(0, min(1, (metrics['win_rate'] - 0.4) / 0.3))
    normalized['sharpe'] = max(0, min(1, metrics['sharpe'] / 3.0))
    normalized['mdd'] = max(0, min(1, 1 - abs(metrics['mdd']) / 0.3))
    normalized['volume'] = max(0, min(1, metrics['volume'] / 100))
    normalized['recency'] = metrics['recency']  # 이미 0~1 (EMA 가중)
    
    score = sum(normalized[k] * weights.get(k, 0) for k in normalized)
    return score

# 예시 가중치 (Copytrade 관점)
weights = {
    'win_rate': 0.20,   # 승률: 기본이지만 과신 금지
    'sharpe': 0.25,     # 샤프: 리스크-조정 성과가 핵심
    'mdd': 0.20,        # MDD: 최대 손실 허용치
    'volume': 0.15,     # 거래량: 표본 크기
    'recency': 0.20,    # 최근성: 과거 성과의 현재 유효성
}
```

### 종합 점수 해석 가이드

| 점수대 | 해석 | 액션 |
|--------|------|------|
| 0.8+ | 검증된 알파 생성기 | 높은 복제 비중(3~5%) |
| 0.6~0.8 | 양호한 성과 | 표준 비중(1~2%) |
| 0.4~0.6 | 평균 수준 | 샘플링 기간 연장 후 재평가 |
| <0.4 | 미검증 or 성과 부진 | 팔로우 제외 또는 최소 비중 |

## 예측 마켓 특화: 지표 보정

일반 금융 지표와 달리 prediction market은 다음과 같은 특성이 있다:

1. **양극화된 수익 분포**: Yes/No 토큰은 0 또는 1로 정산되므로 중간값이 없다. 따라서 변동성 계산 시 outlier 영향이 큼.
2. **이분법적 결과**: 각 트레이드는 full win 또는 full loss이므로 연속성 가정이 깨진다. → **Laplace smoothing** 적용: `(승리 + 1) / (총거래 + 2)`
3. **마켓 임박성 효과**: 이벤트 마감 직전의 포지션은 변동성이 인위적으로 확대됨. 홀딩 타임이 짧은 트레이더의 MDD는 과소평가될 수 있다.
4. **실행 비용 왜곡**: 고빈도 트레이더의 ROI는 슬리피지와 수수료를 반영하지 않으면 과대평가된다. `adjusted_roi = raw_roi - (거래빈도 × 평균슬리피지 × 포지션사이즈)`

## 실제 적용 사례

### 시나리오: 50명 리더보드 → 상위 5명 선별

```
1단계: 필터링 (하위 30명 제거)
  - 30일 거래수 < 10 → 제외
  - MDD > 25% → 제외

2단계: 지표 계산 (잔여 20명)
  - win_rate, sharpe, mdd, volume, recency 계산
  - z-score 표준화

3단계: 가중합 랭킹
  - 상위 5명 선정
  - 상관관계 체크: r > 0.7 쌍 있으면 1명만 선택

4단계: 사이징
  - 종합점수 비례로 자본 할당
  - 개별 트레이더 상한: 5%
```
