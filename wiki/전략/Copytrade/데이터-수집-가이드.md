# Copytrade 데이터 수집 가이드

> Copytrade 전략 수립을 위해 필요한 데이터 소스와 수집/저장/검증 방법을 정리한 가이드입니다.

## 내용

1. 목적

- 리더보드, 포지션, 거래내역 등 트레이더 행동 데이터를 수집하여
  - 카피 후보 선별
  - 전략 패턴 분석
  - 백테스트 및 리스크 모델링

2. 우선 수집 대상 (우선순위)

- 리더보드(API): 상위 트레이더 목록 및 순위 변화
- 프로필(API): 사용자명, 소셜 링크(트위터 등)
- 포지션 및 거래내역(API): 현재 보유 포지션, 과거 트레이드 히스토리
- Top Holders: 특정 마켓 내 대형 보유자 분포
- 외부 텍스트 소스: 트위터(X) 타임라인, Reddit 토론, 블로그 기사

3. 수집 주기 및 보관

- 리더보드: 최소 일간 수집(하루 1회) — 변동성 큼
- 포지션/거래내역: 시간당 또는 6시간 간격(중요 마켓 모니터링 시 단축)
- 외부 텍스트: 새로운 신호 발견 시 스팟 수집(이벤트 기반)
- raw/fetched/에 원본 md로 저장(파일명: YYYY-MM-DD-소스-주제.md)

4. 형식 및 메타데이터

- 각 수집 파일은 YAML frontmatter 포함
  - source, fetched, endpoint, type
- 데이터는 원본 보존(절대 삭제 금지)

5. 관련성 필터링 (Relevance Gate)

- 수집 데이터는 아래 기준 중 1개 이상 충족해야 저장
  - Polymarket 플랫폼 직접 관련 정보
  - Copytrade 전략에 활용 가능한 트레이더 행동 데이터
  - 리더보드 상위 트레이더의 포지션/승률/패턴

6. 자동화 권장 플로우

- raw/sources.md를 읽어 수집 대상 결정
- API → curl/batch 요청 → JSON → 마크다운 변환
- 변환된 md를 raw/fetched/에 저장
- 새 소스 발견 시 raw/sources.md에 추가
- 다음 사이클에서 wiki 업데이트에 반영

7. 품질 관리

- 실패한 API 호출은 로그에 기록하고 건너뜀
- Rate limit 준수(연속 호출 방지)
- 인증 필요 API는 기본적으로 스킵
