# Copytrade 전략분석

> 한 줄 요약: 리더보드 기반 트레이더 선별과 실전 포지션 사이징, 실행 리스크 보정 및 운영 규칙을 제시한다.

## 내용

리더보드에서 성과가 높은 트레이더를 식별하고, 그들의 포지션 패턴과 진입/청산 시점을 분석해 자동 팔로우 전략을 세운다. 리스크 관리는 포지션 크기 제한, 분산(마켓·결과), Negative-Risk 상황 파악으로 수행한다.

### 포지션 사이징 (구체화)

원칙: 각 트레이드에서 잃을 수 있는 금액(리스크)을 먼저 정하고, 그에 맞춰 포지션 크기를 계산한다. 일반 권장 범위는 계정 가치의 0.5~2%.

공식(단순화):
- 허용 손실 L = 계정 가치 * 리스크 비율
- 목표 포지션 P = L / (진입가 대비 손절가 비율 + 예상 거래비용)

예: 계정 1,000 USDC, 리스크 1% → L = 10 USDC. 진입가 대비 손절가 2% (0.02), 예상 거래비용(슬리피지+수수료) 0.5% (0.005) => P = 10 / (0.02 + 0.005) = 10 / 0.025 = 400 USDC

- 이 예에서 원시 계산(500 USDC)보다 보수적으로 포지션을 줄여 400 USDC로 설정한다.
- 만약 오더북 영향으로 추가 슬리피지(예: 0.5%)가 예상되면 비용 항목에 포함시켜 더 축소한다.

리스크 배분 팁:
- 다수 트레이더 분산: 개별 트레이더 노출을 총 자산의 5~20%로 제한
- 마켓·결과 분산: 같은 이벤트의 다수 결과(Yes/No)에는 동시에 노출을 제한
- 레버리지 사용 금지 또는 엄격 제한: Polymarket 구조상 레버리지 관련 예외 상황 존재

### 실전 포지션 계산 — 단계별 예시

1) 계정 가치(E)와 리스크 비율(r)을 정한다. 예: E=10,000 USDC, r=0.01 (1%) → L=100 USDC
2) 트레이더의 진입가 추정(A)과 손절가 비율(s)을 산정(예: s=0.03)
3) 예상 거래비용(c) = 슬리피지 추정 + 수수료 (예: 0.007)
4) P = L / (s + c)

예: E=10,000, r=0.01 → L=100; s=0.03, c=0.007 → P = 100 / 0.037 = 2702 USDC

설명: 이 값이 곧바로 주문 가능한 최대 포지션이다. 추가 안전계층으로 P의 70~90%로 실제 주문을 제한한다.

### 계산 스크립트 (Python 예제)

```python
# 단순 포지션 계산 예제
E = 10000  # 계정 가치
r = 0.01   # 리스크 비율
L = E * r
s = 0.03   # 손절가 비율
c = 0.007  # 예상 거래비용
P_raw = L / (s + c)
P = P_raw * 0.85  # 안전 계수 0.85 적용
print(f"추천 포지션(USDC): {P:.2f}")
```

> 주의: 이 스크립트는 예시일 뿐이며, 오더북 영향, 마켓 유동성, 시장 변동성에 따라 추가 보정이 필요하다.

### 슬리피지·수수료 보정

실행 전 추정치 수집:
1. 과거 유사 주문의 평균 체결 가격 대비 기대 슬리피지 계산
2. 주문 크기별 체결 분포(오더북 깊이) 확인
3. 거래소/플랫폼 수수료 구조 반영

실제 적용: 포지션 산정 시 슬리피지와 수수료를 합쳐 '예상 거래비용'으로 더한다. 긴급 대응 시 시장가 사용은 비용을 증가시키므로, 긴급성 판단 기준을 엄격히 둔다.

### 주문 분할 전략

- 목표 포지션 P를 n개의 리밋 오더로 분할 (예: n=5~10)
- 시간 분산: VWAP 스타일로 분할하여 시간에 따른 가격 변동 리스크를 낮춘다
- 볼륨 민감도: 오더북의 상위 depth가 작을수록 더 작은 분할 단위를 사용

간단한 분할 알고리즘:
1) P_total을 목표로 설정
2) 각 오더량 = P_total / n
3) 각 오더 제출 전 오더북 재확인 → 예상 슬리피지 업데이트 → 필요 시 재계산

### 실행 체크리스트 (강화)

1. 트레이더 이벤트 신뢰성 확인: 리더보드·포지션·트레이드 이벤트의 타임스탬프와 일관성 검사
2. 오더북 깊이(유동성) 점검: 각 분할 주문의 예상 체결률 평가
3. 슬리피지/수수료 추정 반영: P 계산에 포함
4. 주문 분할 계획 수립: n개의 리밋 오더로 분할, 시간 간격 설정
5. 모니터링 룰 설정: 연속 손실, 단일 손실 한도, 슬리피지 임계치
6. 중단 규칙(휴지기): 리더의 전략 변화 의심 시 자동 중단
7. 백테스트/파일럿 승인: 자동화 전 소규모 실거래 검증

### 모니터링 및 중단 규칙(구체값 제안)

- 연속 손실 3회 → 해당 트레이더에 대한 추종 중단 및 재평가
- 단일 트레이드 손실 > 2% of equity → 즉시 해당 포지션 청산 및 추종 중단
- 예상 슬리피지보다 실제 슬리피지 2배 이상 발생 → 추종 일시 중단
- 트레이더 포지션 변화 빈도 급증(평균 대비 3배) → 전략 변경 의심, 휴지기

### 성과 측정

- Gap Attribution: 원 트레이더 수익 vs 카피 트레이더 수익 차이 추적
- 롤링 승률, 샤프, MDD 측정: 주기별(주/월) 리포트
- 실행 비용 모니터: 슬리피지·수수료 합산 비용 추적

## 운영 노트

- 자동화 전 반드시 소규모 실거래(파일럿)로 검증
- 백테스트 결과와 실거래 결과의 차이를 설명하는 로그(분단위 타임스탬프 포함)를 남길 것
- Negative-Risk 상황(자금 투입이 필요한 구조 등)은 레시피에서 제외
