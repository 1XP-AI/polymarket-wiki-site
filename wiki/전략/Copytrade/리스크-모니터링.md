# 리스크 모니터링(Copytrade)

> 한 줄 요약: Copytrade 포트폴리오의 실시간·주기적 리스크 지표를 정의하고 모니터링 절차를 제시한다.

## 내용

Copytrade 운영에서 리스크 모니터링은 단순 손실 추적을 넘어서, 트레이더별·마켓별·전략별로 위험 노출을 정량화하고 이상 징후를 조기에 탐지하는 체계다. 본 문서는 Polymarket에서 팔로우(복제)하는 트레이더의 활동을 관찰해 다음 항목을 지속적으로 계산하도록 권장한다.

- 트레이더별 포지션 집중도: 상위 N개 포지션에 포트폴리오 자본의 몇 %가 몰려있는지
- 마켓별 유동성/슬리피지 예상치: 주문 크기 대비 예상 가격영향
- 손실 트리거(Stop-loss) 비율: 포지션별 손실 임계값 도달률
- 리더보드 변동성 지표: 리더보드 순위 급변(예: 5일 내 상위 10위 변동 횟수)
- 실행지연(Delay) 측정: 트레이더 포지션 발생 시점과 복제 주문 체결 시점의 평균 지연

이들 지표는 raw/fetched API(예: Polymarket 이벤트/markets 엔드포인트)를 주기적으로 폴링하여 산출한다. 데이터 수집은 매 5분~1시간 주기(전략 민감도에 따라 조정)로 수행하며, 급변 시(예: 시장 충격)에는 실시간 WebSocket 연동을 권장한다[[개념/Polymarket-개요]]

## 적용 사례

1) 포지션 집중도가 40%를 넘는 트레이더 발견 → 자동으로 팔로우 비중을 축소하거나 손실 트리거를 단계적으로 적용
2) 특정 마켓에서 예상 슬리피지가 평소보다 2배 증가 → 해당 마켓의 신규 진입을 일시 중단
3) 리더보드 급상승 트레이더가 다수 포지션을 취함 → 해당 트레이더의 과거 승률/청산 기록을 조회하고 가중치 재계산

## 구현 체크리스트

- raw API에서 포지션/마켓 데이터를 정규화하여 DB에 저장
- 지표 산출 스크립트(정의된 수식 포함)를 자동화하여 일일 보고 생성
- 알림 임계값(슬랙/웹훅) 설정: 경고/심각 단계 구분
- 모니터링 대시보드: 트레이더별 리스크 스코어 시각화

## 모니터링 지표 수학적 정의

### 1. 포지션 집중도 (Concentration Index)

상위 N개 포지션에 할당된 자본 비중을 측정한다.

```python
def concentration_index(positions, top_n=5):
    """허핀달-허쉬만 지수(HHI) 기반 집중도 계산"""
    if not positions:
        return 0
    total = sum(p['size_usd'] for p in positions)
    if total == 0:
        return 0
    shares = sorted([p['size_usd'] / total for p in positions], reverse=True)
    top_n_sum = sum(shares[:top_n])
    hhi = sum(s ** 2 for s in shares)
    return {"top_n_concentration": top_n_sum, "hhi": hhi}

# 해석 가이드:
# HHI < 0.10: 잘 분산됨 (낮은 리스크)
# HHI 0.10~0.25: 보통 (모니터링 필요)
# HHI > 0.25: 고집중 (즉시 review)
```

### 2. 실행 지연 (Execution Delay)

트레이더의 주문 시점과 팔로워의 복제 주문 시점 차이. Polymarket의 경우 시장이 빠르게 움직이므로 지연이 수익률에 직접적 영향을 준다.

```python
def measure_execution_delay(trader_timestamp, follower_timestamp, market_snapshot):
    """
    trader_timestamp: 트레이더가 포지션을 연 시간 (UTC)
    follower_timestamp: 팔로워가 동일 포지션을 복제한 시간 (UTC)
    market_snapshot: 두 시점 사이의 마켓 가격 스냅샷 리스트

    반환: 지연(초), 슬리피지(%), 예상 PnL 손실
    """
    delay_sec = (follower_timestamp - trader_timestamp).total_seconds()
    price_at_trade = market_snapshot[0]['price']
    price_at_copy = market_snapshot[-1]['price']
    slippage_pct = abs(price_at_copy - price_at_trade) / price_at_trade * 100
    return {
        "delay_seconds": delay_sec,
        "slippage_pct": slippage_pct,
        "critical": delay_sec > 30 or slippage_pct > 2.0
    }
```

### 3. 손실 트리거 메트릭 (Drawdown Alert)

```python
# 손실 임계값 설정 예시
DRAWDOWN_THRESHOLDS = {
    "warning": 0.05,     # 5% 손실 → 경고 알림
    "reduce": 0.10,      # 10% 손실 → 팔로우 비중 50% 축소
    "stop": 0.15,        # 15% 손실 → 팔로우 일시 정지
    "review": 0.20       # 20% 손실 → 전무 평가 후 재개여부 결정
}
```

## 알림 규칙 (Alerting Rules)

| 트리거 | 심각도 | 액션 | 채널 |
|--------|--------|------|------|
| HHI > 0.25 | 경고 | 트레이더 리밸런싱 확인 | Slack #risk-alerts |
| 실행 지연 > 30초 | 경고 | API rate limit 확인 | Slack #risk-alerts |
| 손실 5% 초과 | 주의 | 포트폴리오 review 예약 | Slack #risk-alerts |
| 손실 10% 초과 | 심각 | 팔로우 규모 50% 자동 축소 | Slack + Webhook |
| 손실 15% 초과 | 위험 | 팔로우 자동 정지 | Slack + Webhook + 이메일 |
| 슬리피지 > 5% | 경고 | 해당 마켓 진입 일시 중단 | Slack #execution |
| 리더보드 순위 10위 이상 하락 | 주의 | 트레이더 재평가 | Slack #trader-review |
| Neg-Risk 마켓 가격 불균형 감지 | 정보 | [[개념/Negative-Risk]] 기회 탐지 알림 | Slack #arb-opps |

## 주기별 모니터링 스케줄

| 주기 | 대상 | 방법 |
|------|------|------|
| 1분 | 실행 지연, 즉각 손실 | WebSocket 스트리밍, 로컬 카운터 |
| 5분 | 포지션 변동, 슬리피지 | Gamma API 폴링 |
| 1시간 | HHI, 드리프트 분석 | DB 집계, Python 스크립트 |
| 일일 | 종합 리스크 리포트 | 일일 배치 잡, 대시보드 갱신 |
| 주간 | 트레이더 재평가, 전략 review | 수동 분석 + 자동 요약 |
