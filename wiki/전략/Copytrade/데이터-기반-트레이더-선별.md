# 데이터 기반 트레이더 선별

> 한 줄 요약: 리더보드, 포지션 데이터, 소셜 시그널을 결합해 copytrade 후보 트레이더를 정량적으로 선별하는 방법과 체크리스트.

## 배경

Copytrade의 핵심은 '누구를 따라갈 것인가'이다. 정성적 판단만으로는 리스크가 크므로, 리더보드 순위, 포지션 통계, 포지션 지속성, 승률, 소셜(트위터/레딧)에서의 전략 언급 빈도 등 여러 지표를 결합해 정량적 선별을 수행한다.

## 3단계 필터링 파이프라인

```
전체 트레이더 (N)
    │
    ▼
 1차: 유동성 필터  ── 거래량 > $10K, 활동일수 > 30일
    │ (약 80% 제거)
    ▼
 2차: 통계 필터   ── 승률 > 52%, MaxDD < 20%, Sharpe > 0.5
    │ (약 90% 제거)
    ▼
 3차: 알파 검증   ── Bootstrap p-value < 0.05, 일관성 점수
    │ (약 95% 제거)
    ▼
  최종 후보  (2~5명)
```

### 1차 필터 — 기본 유동성

선별 대상에서 제외해야 할 노이즈를 제거한다:

- **최소 거래량**: 최근 30일 누적 거래량 > $10,000. 미달 시 통계적 신뢰도 부족.
- **최소 활동일수**: 계정 생성 후 첫 거래 기준 30일 이상. 신규 계정의 '초반 운'을 피한다.
- **최소 거래 횟수**: 최근 30일 15회 이상. 샘플 크기가 작으면 승률이 왜곡된다.
- **유동성 마켓 비중**: Polymarket Gamma API `/markets`에서 `liquidity > threshold`인 마켓에서만 거래한 비율이 70% 이상. 유동성 부족 마켓에서의 거래는 슬리피지 왜곡으로 실제 성과와 괴리 발생.

### 2차 필터 — 성과 일관성

- **승률(Win Rate)**: 52% 이상. prediction market에서 50%는 무작위 추측과 동일. 52%는 통계적 edge의 최소선.
- **최대 드로우다운(Max Drawdown)**: 20% 이하. 그 이상은 리스크 관리 부재 신호.
- **Sharpe Ratio**: 0.5 이상. 일관된 수익을 내는지 확인.
- **포지션 지속성**: 동일 이벤트 유형(예: 정치, 크립토)에서 3회 이상 연속 베팅 시 평균 승률이 개인 승률과 ±10% 이내. 특정 마켓에만 편중되지 않았는지 확인.
- **평균 포지션 유지시간**: 1시간~7일 범위. 1시간 미만은 HFT 신호(복제 불리), 7일 초과는 스윙/장기(다른 리스크 프로파일).

### 3차 필터 — 알파 검증

Bootstrap 리샘플링과 Laplace smoothing을 적용해 통계적 유의성 확인:

```python
def laplace_smoothed_winrate(wins, total, alpha=1.0):
    """소표본 편향 보정: (wins + α) / (total + 2α)"""
    return (wins + alpha) / (total + 2 * alpha)

def bootstrap_alpha_test(returns, n_bootstrap=10000, threshold=0.05):
    """관측 성과가 랜덤보다 유의미한지 검증"""
    import numpy as np
    observed = np.mean(returns)
    boot_means = []
    for _ in range(n_bootstrap):
        # Null: 중심 0인 랜덤 샘플
        sample = np.random.choice(returns, size=len(returns), replace=True)
        sample = sample - np.mean(sample) + 0  # 중심화
        boot_means.append(np.mean(sample))
    p_value = np.mean(np.array(boot_means) >= observed)
    return p_value < threshold
```

- **Bootstrap p-value**: 0.05 미만이면 통계적으로 유의미한 알파 존재.
- **표본 분할**: 최근 60일 데이터로 train, 최근 30일로 validate. 과적합 방지.
- **Laplace smoothed 승률**: 거래 횟수가 20회 미만인 트레이더는 α=1.0으로 보정. 소표본 과대평가를 방지.

## Follow Score 알고리즘 (종합 점수)

`FollowScore = 0.30 × S(WinRate) + 0.25 × S(Consistency) + 0.20 × S(VolumeStability) - 0.15 × S(MaxDD) - 0.10 × S(CorrelationRisk)`

여기서 `S(·)`는 0~1 구간으로 min-max 정규화한 값. CorrelationRisk는 후보 트레이더와 이미 팔로우 중인 트레이더 간 포지션 동시진행 비율. 0.6 이상이면 중복 노출 경고.

## 소셜 시그널 교차검증

리더보드 수치만으로는 '왜' 그 트레이더가 승리하는지 알 수 없다. 다음 소셜 지표를 보조 지표로 활용한다:

- **전략 언급 빈도**: 트위터/레딧에서 "arbitrage", "negative risk", "CTF" 등 Polymarket 특화 전략 키워드를 실제로 언급하는지.
- **계정 신뢰도**: 팔로워 수, 활동 기간, 과거 발언의 정확성(예측 적중률).
- **주의**: 소셜 증거는 2차 지표로만 사용. 과대해석 시 '셀프 프로모션 잘 하는 loser'를 선택할 수 있다.

## 선별 체크리스트

1. Gamma API `/events` + PolymarketAnalytics에서 상위 5% 트레이더 추출
2. 1차 필터 적용: 유동성/거래량/활동일수 기준 통과자만 남김
3. 2차 필터 적용: 승률 > 52%, MaxDD < 20%, Sharpe > 0.5
4. Bootstrap 알파 검증: p-value < 0.05 확인
5. Laplace smoothed 승률로 소표본 보정
6. CorrelationRisk 계산: 기존 팔로우 대상과 중복도 확인
7. FollowScore 산정 → 임계값(예: 0.65) 이상만 후보로 선정
8. 최종 3명 이하로 압축 → 백테스트(과거 포트폴리오 적용) [[개념/Copytrade#백테스트 설계 (권장)]]

## 리스크 고려사항

- **리더보드 샘플 편향**: 리더보드는 '생존자 편향'이 있음. 이미 탈락한 트레이더는 보이지 않음.
- **마켓 특이성**: 스포츠 마켓 승률은 크립토 마켓과 다른 분포. 카테고리별 별도 평가 권장.
- **정적 vs 동적 가중치**: 초기에는 Equal Weight로 시작, 데이터 누적으로 Laplace α를 줄이고 Kelly Fraction으로 전환. ([[전략/Copytrade/팔로우-스코어링-모델]]의 Sigmoid 변환 참고)
