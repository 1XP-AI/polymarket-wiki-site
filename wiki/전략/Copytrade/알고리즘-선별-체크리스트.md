# 알고리즘 선별 체크리스트

> 자동화/반자동화 환경에서 리더보드 기반 트레이더를 시스템적으로 선별하기 위한 체크리스트

## 한 줄 요약

리더보드 데이터와 온체인/오프체인 신호를 결합해 copytrade 대상으로 적합한 트레이더를 정량적으로 선별하는 절차와 모니터링 포인트를 제시한다.

## 내용

1. 데이터 소스 확인
   - 리더보드(Polymarket), 트랜잭션(온체인), 트위터/Reddit 언급 횟수, API 이벤트 피드 등을 우선 수집
   - 수집 주기와 신선도 기준 설정(예: 24시간 이내 활동)

2. 기본 필터
   - 활동 기간: 최소 N주 이상(예: 4주)
   - 최소 거래 횟수: 최근 M건 이상(예: 10건)
   - 포지션 변동성: 지나치게 잦은 진입/청산은 제외

3. 성과 및 안정성 지표
   - 누적 수익률(RoI), 승률(Win Rate), 평균 수익/손실 비율
   - Max Drawdown(기간별 최대 손실) 계산
   - 포지션 보유 기간의 중앙값/평균

4. 리스크 적합성(Keep-out 규칙)
   - 큰 레버리지/베팅 집중(상위 X% 포지션 집중)은 제외 대상
   - 특정 시장(예: 저유동성 마켓)에만 집중하는 트레이더는 경고

5. 행태 분석(전략적 신호)
   - 동일한 패턴 반복 여부(예: 특정 이벤트 직후 매수 반복)
   - 뉴스/트윗 연동: 소셜 신호가 거래에 영향 미치는지 상관 분석

6. 실행 비용 추정
   - 평균 슬리피지(과거 체결 데이터 기반) 및 수수료 추정
   - 최소 복제 단위(포지션 사이즈)와 시장 영향 비용 계산

7. 자동화 적합성 점수화
   - 항목별 가중치(예: 성과 40%, 안정성 30%, 실행비용 20%, 소셜 신호 10%)로 합산
   - 특정 임계값(예: 0.7 이상)일 때 자동 팔로우 후보로 표기

8. 모니터링 및 탈락 규칙
   - 실시간 알림: Drawdown > threshold, 연속 손실 n회 등
   - 주기적 재평가: 7일/30일 단위로 점수 재계산

## 예시

- Theo와 같은 상위 트레이더를 예시로, 최근 30일 승률 65%, 평균 보유기간 3일, 평균 슬리피지 0.3% → 자동화 후보(점수 0.78)
