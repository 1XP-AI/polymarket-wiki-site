### 마켓 유형

#### 1. 경기 결과 (Win/Lose)
가장 기본적인 Binary 마켓.

```
Will [Team A] beat [Team B]?
→ Yes / No
```

#### 2. 스프레드 마켓
점수 차이에 대한 베팅.

```
Will [Team A] beat [Team B] by more than X points?
→ Yes: Team A가 X점 이상 차이로 승리
→ No: Team A가 X점 이하 차이이거나 패배
```

#### 3.ournament 우승
시즌 전체 결과 예측.

```
Will Carolina Hurricanes win the 2026 NHL Stanley Cup?
→ Yes 11.5% / No 88.5% (vol $253K)
```

## 스포츠 마켓 특징

### 장점
- **결과 판정이 빠름**: 경기 종료 후 수小时内 확정
- **객관적 결과**: 판정 불확실성이 낮음
- **계절적 관심**: NFL 시즌, NBA 플레이오프 등에 거래량 급증

### 단점
- **경기前一天/당일 정보**: 사전 정보 기반 베팅 가능
- **부상/라인업 변경**: 경기 直前に 확률 급변
- **시장 비효율성**: 팬 기반 거래로 비합리적 가격 형성 가능

## 가격 형성 분석

### 스프레드 마켓 해석

예시: "Will Team A beat Team B by more than 5.5 points?"
- Yes $0.60 = Team A가 6점 이상 차이로 승리할 확률 60%
- Spread 마켓은 **핸디캡** 개념과 유사

###ournament 마켓

```
Carolina Hurricanes Yes: $0.115 (11.5%)

```

## 트레이딩 전략

### 전략 1: 라인업 분석
선수 출전 정보를 기반으로 사전에 확률 변화 예측:
- [[기술/API/Polymarket-API-레퍼런스]]의 마켓 API로 실시간 마켓 모니터링
- 사전 정보 공개 시 → 가격 급변 → 진입 기회

### 전략 2: 실시간 Copytrade
경기 중에 트레이더의 베팅 패턴 추적:
- [[전략/Copytrade/자동화-플로우]]의 WebSocket 활용
- [[기술/WebSocket-실시간데이터]]로 실시간 가격 변동 감지

### 전략 3: 유동성 활용
스포츠 마켓은 **경기 전날**에 유동성이 가장 높음:
- 마켓이 불안정할 때 진입 → 결과逼近 시 수익 실현

## 마켓 분석 도구

### API로 스포츠 마켓 필터링

```bash
# NBA 마켓만 조회
curl -s "https://gamma-api.polymarket.com/public-search?q=nba&type=markets"

# 마켓 상세
curl -s "https://gamma-api.polymarket.com/markets/{id}"
```

### WebSocket으로 실시간 모니터링

```javascript
// 특정 마켓의 실시간 가격 변동 감지
ws.on('message', (data) => {
  const update = JSON.parse(data);
  if (update.type === 'price_change') {
    console.log(`Price: ${update.price}, Volume: ${update.volume}`);
  }
});
```
